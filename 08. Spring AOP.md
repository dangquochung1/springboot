## 1. AOP là gì?

### 1.1. Khái niệm

__AOP (Aspect-Oriented Programming)__ — Lập trình hướng khía cạnh — là một phương pháp lập trình cho phép tách các __cross-cutting concerns__ (mối quan tâm xuyên suốt) ra khỏi business logic chính.

__Cross-cutting concerns__ là các chức năng cần thiết ở nhiều nơi trong ứng dụng nhưng không thuộc về business logic:
- __Logging__ (Ghi log)
- __Security__ (Kiểm tra quyền)
- __Transaction Management__ (Quản lý giao dịch)
- __Caching__ (Bộ nhớ đệm)
- __Performance Monitoring__ (Giám sát hiệu suất)
- __Error Handling__ (Xử lý lỗi)

Không dùng AOP — code bị trộn lẫn:
```java
public class UserService {
    public User findById(Long id) {
        log.info("Finding user by id: " + id);           // Logging
        checkPermission("USER_READ");                      // Security
        long start = System.currentTimeMillis();           // Performance
        User user = userRepository.findById(id);           // Business Logic
        log.info("Found user in " + (System.currentTimeMillis() - start) + "ms");
        return user;
    }
}
```

Dùng AOP — code sạch hơn:
```java
public class UserService {
    public User findById(Long id) {
        return userRepository.findById(id);  // Chỉ business logic
    }
}
// Logging, Security, Performance → xử lý ở Aspect riêng
```

<br />

### 1.2. Các thuật ngữ AOP

| Thuật ngữ | Mô tả | Ví dụ |
| --------- | ----- | ----- |
| __Aspect__ | Module chứa cross-cutting concern | LoggingAspect, SecurityAspect |
| __Advice__ | Code được thực thi tại một thời điểm cụ thể | Log trước khi method chạy |
| __Join Point__ | Điểm trong chương trình nơi Advice có thể can thiệp | Mỗi lời gọi method |
| __Pointcut__ | Biểu thức xác định __nơi nào__ Advice được áp dụng | "Tất cả method trong package service" |
| __Target Object__ | Đối tượng bị AOP can thiệp | UserService instance |
| __Proxy__ | Đối tượng bọc ngoài target để áp dụng Advice | JDK Dynamic Proxy hoặc CGLIB |
| __Weaving__ | Quá trình liên kết Aspect với target | Compile-time hoặc Runtime |

## 2. Các loại Advice

### 2.1. Tổng quan

| Advice | Thời điểm thực thi |
| ------ | ------------------- |
| `@Before` | __Trước__ khi method chạy |
| `@After` | __Sau__ khi method chạy (luôn luôn, dù có exception) |
| `@AfterReturning` | Sau khi method __trả về thành công__ |
| `@AfterThrowing` | Sau khi method __ném exception__ |
| `@Around` | __Bao quanh__ method (trước + sau), kiểm soát hoàn toàn |

<br />

### 2.2. Ví dụ: Logging Aspect

```java
@Aspect
@Component
@Slf4j
public class LoggingAspect {

    // @Before — Chạy TRƯỚC method
    @Before("execution(* com.example.myapp.service.*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        String methodName = joinPoint.getSignature().getName();
        Object[] args = joinPoint.getArgs();
        log.info(">>> Calling: {}() with args: {}", methodName, Arrays.toString(args));
    }

    // @AfterReturning — Chạy SAU khi method trả về thành công
    @AfterReturning(
            pointcut = "execution(* com.example.myapp.service.*.*(..))",
            returning = "result")
    public void logAfterReturning(JoinPoint joinPoint, Object result) {
        log.info("<<< {}() returned: {}", joinPoint.getSignature().getName(), result);
    }

    // @AfterThrowing — Chạy khi method ném exception
    @AfterThrowing(
            pointcut = "execution(* com.example.myapp.service.*.*(..))",
            throwing = "exception")
    public void logAfterThrowing(JoinPoint joinPoint, Throwable exception) {
        log.error("!!! {}() threw: {}", joinPoint.getSignature().getName(),
                exception.getMessage());
    }

    // @Around — Bao quanh method (mạnh nhất)
    @Around("execution(* com.example.myapp.service.*.*(..))")
    public Object logAround(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        String methodName = joinPoint.getSignature().getName();

        log.info(">>> Start: {}()", methodName);
        try {
            Object result = joinPoint.proceed();  // Gọi method gốc
            long duration = System.currentTimeMillis() - start;
            log.info("<<< End: {}() - {}ms", methodName, duration);
            return result;
        } catch (Throwable e) {
            log.error("!!! Error in {}(): {}", methodName, e.getMessage());
            throw e;
        }
    }
}
```

## 3. Pointcut Expressions

### 3.1. Cú pháp

```
execution(modifiers? return-type declaring-type? method-name(params) throws?)
```

### 3.2. Ví dụ

```java
// Tất cả method trong package service
@Pointcut("execution(* com.example.myapp.service.*.*(..))")
public void serviceLayer() {}

// Tất cả method public
@Pointcut("execution(public * *(..))")
public void publicMethods() {}

// Method có tên bắt đầu bằng "find"
@Pointcut("execution(* com.example.myapp.repository.*.find*(..))")
public void findMethods() {}

// Method có annotation @Transactional
@Pointcut("@annotation(org.springframework.transaction.annotation.Transactional)")
public void transactionalMethods() {}

// Tất cả class có annotation @Service
@Pointcut("@within(org.springframework.stereotype.Service)")
public void serviceBeans() {}

// Kết hợp nhiều pointcut
@Before("serviceLayer() && publicMethods()")
public void logPublicServiceMethods(JoinPoint joinPoint) { ... }
```

## 4. Custom Annotation với AOP

### 4.1. Tạo Custom Annotation

Kết hợp AOP với custom annotation để tạo tính năng tái sử dụng:

__LogExecutionTime.java__
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface LogExecutionTime {
}
```

__PerformanceAspect.java__
```java
@Aspect
@Component
@Slf4j
public class PerformanceAspect {

    @Around("@annotation(com.example.myapp.annotation.LogExecutionTime)")
    public Object logExecutionTime(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        Object result = joinPoint.proceed();
        long duration = System.currentTimeMillis() - start;

        log.info("{}() executed in {}ms",
                joinPoint.getSignature().getName(), duration);
        return result;
    }
}
```

Sử dụng:
```java
@Service
public class ReportService {

    @LogExecutionTime  // Tự động log thời gian thực thi
    public Report generateReport(Long userId) {
        // Complex business logic
        return report;
    }
}
```

Output:
```
INFO  - generateReport() executed in 1523ms
```

## 5. Spring Caching

### 5.1. Caching là gì?

__Caching__ lưu trữ kết quả của các thao tác tốn thời gian vào bộ nhớ đệm để tái sử dụng, giúp __cải thiện performance__ đáng kể.

Spring Cache Abstraction được xây dựng trên nền tảng AOP.

<br />

### 5.2. Kích hoạt Caching

```java
@SpringBootApplication
@EnableCaching
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

<br />

### 5.3. Cache Annotations

| Annotation | Mô tả |
| ---------- | ----- |
| `@Cacheable` | Lưu kết quả vào cache, lần sau trả về từ cache |
| `@CachePut` | Luôn thực thi method và cập nhật cache |
| `@CacheEvict` | Xóa entry khỏi cache |
| `@Caching` | Kết hợp nhiều cache annotations |

```java
@Service
@RequiredArgsConstructor
public class ProductService {

    private final ProductRepository productRepository;

    // Lần đầu: query DB và lưu vào cache "products"
    // Lần sau: trả về từ cache, KHÔNG query DB
    @Cacheable(value = "products", key = "#id")
    public Product findById(Long id) {
        log.info("Fetching product from DB: {}", id);
        return productRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("Product not found"));
    }

    // Luôn thực thi và cập nhật cache
    @CachePut(value = "products", key = "#product.id")
    public Product update(Product product) {
        return productRepository.save(product);
    }

    // Xóa entry khỏi cache khi xóa product
    @CacheEvict(value = "products", key = "#id")
    public void delete(Long id) {
        productRepository.deleteById(id);
    }

    // Xóa toàn bộ cache
    @CacheEvict(value = "products", allEntries = true)
    public void clearCache() {
        log.info("Cache cleared");
    }
}
```

### 5.4. Cấu hình Cache Provider

Mặc định Spring dùng __ConcurrentHashMap__ (in-memory). Cho production, nên dùng __Redis__ hoặc __Caffeine__.

Redis:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-redis</artifactId>
</dependency>
```

```properties
spring.cache.type=redis
spring.data.redis.host=localhost
spring.data.redis.port=6379
spring.cache.redis.time-to-live=3600000
```

## 6. Câu hỏi phỏng vấn thường gặp

__Q: AOP hoạt động như thế nào trong Spring?__

Spring AOP sử dụng __proxy pattern__. Khi Bean được tạo, Spring tạo một proxy bọc ngoài Bean gốc. Khi method được gọi, proxy thực thi Advice trước/sau method gốc. Spring dùng __JDK Dynamic Proxy__ (cho interface) hoặc __CGLIB__ (cho class).

__Q: `@Around` khác gì các Advice khác?__

`@Around` là Advice mạnh nhất, có thể: (1) quyết định có gọi method gốc hay không, (2) thay đổi tham số, (3) thay đổi kết quả trả về, (4) bắt exception. Các advice khác chỉ can thiệp ở một thời điểm.

__Q: `@Cacheable` vs `@CachePut`?__

`@Cacheable` kiểm tra cache trước, nếu có → trả về từ cache, __không gọi method__. `@CachePut` luôn __gọi method__ rồi cập nhật cache. Dùng `@CachePut` cho update operations.
