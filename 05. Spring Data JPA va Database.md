# Spring Data JPA và Database

<br />

## 1. Spring Data JPA

### 1.1. Spring Data JPA là gì?

__Spring Data JPA__ là một module của __Spring Data__ giúp đơn giản hóa việc tương tác với cơ sở dữ liệu quan hệ thông qua __JPA (Java Persistence API)__. Nó giảm thiểu đáng kể lượng __boilerplate code__ cần viết cho tầng data access.

Spring Data JPA cung cấp:
- __Repository abstraction__: Interface sẵn có với các phương thức CRUD.
- __Derived Query Methods__: Tự động tạo query từ tên method.
- __@Query annotation__: Viết JPQL hoặc Native SQL trực tiếp.
- __Pagination & Sorting__: Hỗ trợ phân trang và sắp xếp sẵn.
- __Auditing__: Tự động tracking ngày tạo, ngày sửa.

<br />

### 1.2. Cấu hình Database

__application.properties__
```properties
# MySQL
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=password
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.MySQLDialect
```

Các giá trị của `spring.jpa.hibernate.ddl-auto`:

| Giá trị | Mô tả |
| ------- | ----- |
| `none` | Không làm gì (production) |
| `validate` | Chỉ kiểm tra schema khớp với entity |
| `update` | Cập nhật schema (thêm cột/bảng mới, không xóa) |
| `create` | Xóa và tạo lại schema mỗi lần khởi động |
| `create-drop` | Như `create` nhưng xóa schema khi shutdown |

> __Best Practice__: Dùng `update` cho development, `validate` hoặc `none` cho production. Sử dụng migration tool như __Flyway__ hoặc __Liquibase__ cho production.

## 2. Entity

### 2.1. Tạo Entity

```java
@Getter
@Setter
@NoArgsConstructor
@AllArgsConstructor
@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String name;

    @Column(nullable = false, unique = true)
    private String email;

    @Column(nullable = false)
    private String password;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private Role role;

    @Column(name = "created_at", updatable = false)
    @CreationTimestamp
    private LocalDateTime createdAt;

    @Column(name = "updated_at")
    @UpdateTimestamp
    private LocalDateTime updatedAt;
}
```

<br />

### 2.2. Entity Annotations

| Annotation | Mô tả |
| ---------- | ----- |
| `@Entity` | Đánh dấu class là một JPA Entity |
| `@Table(name)` | Chỉ định tên bảng trong database |
| `@Id` | Đánh dấu primary key |
| `@GeneratedValue` | Chiến lược sinh ID tự động |
| `@Column` | Cấu hình cột (name, nullable, unique, length) |
| `@Enumerated` | Ánh xạ Enum (STRING hoặc ORDINAL) |
| `@CreationTimestamp` | Tự động set thời gian tạo |
| `@UpdateTimestamp` | Tự động set thời gian cập nhật |
| `@Transient` | Không ánh xạ vào database |

<br />

### 2.3. Entity Relationships

| Annotation | Quan hệ | Ví dụ |
| ---------- | ------- | ----- |
| `@OneToOne` | 1 - 1 | User ↔ UserProfile |
| `@OneToMany` | 1 - N | User → List\<Order\> |
| `@ManyToOne` | N - 1 | Order → User |
| `@ManyToMany` | N - N | Student ↔ Course |

```java
@Entity
@Table(name = "orders")
public class Order {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "order_number", unique = true)
    private String orderNumber;

    private BigDecimal totalAmount;

    // Many Orders belong to One User
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id", nullable = false)
    private User user;

    // One Order has Many OrderItems
    @OneToMany(mappedBy = "order", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<OrderItem> items = new ArrayList<>();
}
```

> __Lưu ý về Fetch Type__:
> - `FetchType.LAZY` (mặc định cho `@OneToMany`, `@ManyToMany`): Chỉ load khi truy cập.
> - `FetchType.EAGER` (mặc định cho `@ManyToOne`, `@OneToOne`): Load ngay cùng entity cha.
> - __Best Practice__: Luôn dùng `LAZY` và fetch khi cần qua __JPQL JOIN FETCH__ để tránh __N+1 problem__.

## 3. Repository

### 3.1. Repository Interface Hierarchy

```
Repository (marker)
   └── CrudRepository (CRUD methods)
          └── PagingAndSortingRepository (+ pagination, sorting)
                 └── JpaRepository (+ JPA specific methods)
```

| Interface | Các method chính |
| --------- | --------------- |
| `CrudRepository` | `save()`, `findById()`, `findAll()`, `delete()`, `count()`, `existsById()` |
| `PagingAndSortingRepository` | + `findAll(Sort)`, `findAll(Pageable)` |
| `JpaRepository` | + `flush()`, `saveAndFlush()`, `deleteInBatch()`, `findAll()` returns List |

<br />

### 3.2. Tạo Repository

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    // JpaRepository đã cung cấp sẵn các method CRUD
    // Không cần viết implementation
}
```

Sử dụng:
```java
@Service
@RequiredArgsConstructor
public class UserService {

    private final UserRepository userRepository;

    // Tìm tất cả
    public List<User> findAll() {
        return userRepository.findAll();
    }

    // Tìm theo ID
    public User findById(Long id) {
        return userRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("User not found"));
    }

    // Lưu mới hoặc cập nhật
    public User save(User user) {
        return userRepository.save(user);
    }

    // Xóa
    public void delete(Long id) {
        userRepository.deleteById(id);
    }

    // Kiểm tra tồn tại
    public boolean exists(Long id) {
        return userRepository.existsById(id);
    }
}
```

## 4. Query Methods

### 4.1. Derived Query Methods

Spring Data JPA tự động tạo query SQL từ __tên method__ dựa trên quy tắc đặt tên.

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // SELECT * FROM users WHERE email = ?
    Optional<User> findByEmail(String email);

    // SELECT * FROM users WHERE name LIKE '%keyword%'
    List<User> findByNameContaining(String keyword);

    // SELECT * FROM users WHERE role = ? AND active = true
    List<User> findByRoleAndActiveTrue(Role role);

    // SELECT * FROM users WHERE age BETWEEN ? AND ?
    List<User> findByAgeBetween(int min, int max);

    // SELECT * FROM users WHERE email = ? (kiểm tra tồn tại)
    boolean existsByEmail(String email);

    // SELECT COUNT(*) FROM users WHERE role = ?
    long countByRole(Role role);

    // SELECT * FROM users ORDER BY created_at DESC
    List<User> findAllByOrderByCreatedAtDesc();

    // DELETE FROM users WHERE active = false
    void deleteByActiveFalse();

    // SELECT * FROM users WHERE name = ? OR email = ?
    List<User> findByNameOrEmail(String name, String email);

    // SELECT TOP 5 FROM users ORDER BY created_at DESC
    List<User> findTop5ByOrderByCreatedAtDesc();
}
```

Các keyword hỗ trợ:

| Keyword | Ví dụ | SQL tương đương |
| ------- | ----- | --------------- |
| `And` | `findByNameAndEmail` | `WHERE name = ? AND email = ?` |
| `Or` | `findByNameOrEmail` | `WHERE name = ? OR email = ?` |
| `Between` | `findByAgeBetween` | `WHERE age BETWEEN ? AND ?` |
| `LessThan` | `findByAgeLessThan` | `WHERE age < ?` |
| `GreaterThan` | `findByAgeGreaterThan` | `WHERE age > ?` |
| `Like` | `findByNameLike` | `WHERE name LIKE ?` |
| `Containing` | `findByNameContaining` | `WHERE name LIKE %?%` |
| `In` | `findByRoleIn` | `WHERE role IN (?, ?)` |
| `OrderBy` | `findByOrderByNameAsc` | `ORDER BY name ASC` |
| `Not` | `findByNameNot` | `WHERE name <> ?` |
| `IsNull` | `findByEmailIsNull` | `WHERE email IS NULL` |
| `True/False` | `findByActiveTrue` | `WHERE active = true` |

<br />

### 4.2. `@Query` Annotation — JPQL

Khi query phức tạp hơn, sử dụng __JPQL__ (Java Persistence Query Language):

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // JPQL — truy vấn trên Entity (không phải table)
    @Query("SELECT u FROM User u WHERE u.email = :email")
    Optional<User> findUserByEmail(@Param("email") String email);

    @Query("SELECT u FROM User u WHERE u.name LIKE %:keyword% OR u.email LIKE %:keyword%")
    List<User> searchUsers(@Param("keyword") String keyword);

    @Query("SELECT u FROM User u JOIN u.orders o WHERE o.totalAmount > :amount")
    List<User> findUsersWithOrdersAbove(@Param("amount") BigDecimal amount);

    // Update query
    @Modifying
    @Query("UPDATE User u SET u.active = false WHERE u.lastLogin < :date")
    int deactivateInactiveUsers(@Param("date") LocalDateTime date);
}
```

<br />

### 4.3. `@Query` Annotation — Native SQL

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // Native SQL — truy vấn trực tiếp trên table
    @Query(value = "SELECT * FROM users WHERE email = :email", nativeQuery = true)
    Optional<User> findByEmailNative(@Param("email") String email);

    @Query(value = "SELECT u.* FROM users u " +
            "INNER JOIN orders o ON u.id = o.user_id " +
            "GROUP BY u.id HAVING COUNT(o.id) > :minOrders",
            nativeQuery = true)
    List<User> findActiveCustomers(@Param("minOrders") int minOrders);
}
```

> __JPQL vs Native SQL__: Ưu tiên dùng __JPQL__ vì nó __database-independent__. Chỉ dùng __Native SQL__ khi cần tính năng đặc thù của database (stored procedure, window functions,...).

## 5. Pagination và Sorting

### 5.1. Pagination

```java
@Service
@RequiredArgsConstructor
public class UserService {

    private final UserRepository userRepository;

    public Page<User> getUsers(int page, int size) {
        Pageable pageable = PageRequest.of(page, size);
        return userRepository.findAll(pageable);
    }
}
```

```java
@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @GetMapping
    public ResponseEntity<Page<User>> getUsers(
            @RequestParam(defaultValue = "0") int page,
            @RequestParam(defaultValue = "10") int size) {
        return ResponseEntity.ok(userService.getUsers(page, size));
    }
}
```

## 6. Auditing

### 6.1. Tự động tracking ngày tạo, ngày sửa

Kích hoạt JPA Auditing:
```java
@Configuration
@EnableJpaAuditing
public class JpaConfig {
}
```

Tạo __BaseEntity__:
```java
@Getter
@Setter
@MappedSuperclass
@EntityListeners(AuditingEntityListener.class)
public abstract class BaseEntity {

    @CreatedDate
    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @LastModifiedDate
    @Column(name = "updated_at")
    private LocalDateTime updatedAt;
}
```

Sử dụng:
```java
@Entity
@Table(name = "users")
public class User extends BaseEntity {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String name;
    private String email;
    // createdAt và updatedAt được kế thừa từ BaseEntity
}
```

## 7. Câu hỏi phỏng vấn thường gặp

__Q: JpaRepository vs CrudRepository?__

`JpaRepository` kế thừa từ `CrudRepository`, cung cấp thêm các method JPA-specific như `flush()`, `saveAndFlush()`, `deleteInBatch()`. `findAll()` của JpaRepository trả về `List` thay vì `Iterable`.

__Q: N+1 Problem là gì?__

Khi load entity cha (1 query), rồi mỗi entity con lại phát sinh thêm 1 query → N+1 queries. Giải pháp: dùng `@EntityGraph` hoặc `JOIN FETCH` trong JPQL.

__Q: JPQL khác SQL như thế nào?__

JPQL truy vấn trên __Entity__ và __property__ (Java objects), SQL truy vấn trên __table__ và __column__ (database). JPQL database-independent.
