## 1. SELECT — Truy vấn dữ liệu

### 1.1. Cú pháp cơ bản

```sql
SELECT column1, column2, ...
FROM table_name
WHERE condition
ORDER BY column ASC|DESC
LIMIT n;
```

Ví dụ:
```sql
-- Lấy tất cả cột
SELECT * FROM employees;

-- Lấy cột cụ thể
SELECT name, email, salary FROM employees;

-- Dùng alias
SELECT name AS "Họ tên", salary AS "Lương" FROM employees;

-- Dùng DISTINCT — loại bỏ trùng lặp
SELECT DISTINCT department FROM employees;
```

<br />

### 1.2. Thứ tự thực thi câu lệnh SQL

SQL __không thực thi theo thứ tự viết__. Thứ tự thực thi thực tế:

| Thứ tự thực thi | Câu lệnh | Chức năng |
| :--------------: | --------- | --------- |
| 1 | `FROM` / `JOIN` | Xác định nguồn dữ liệu |
| 2 | `WHERE` | Lọc hàng trước khi nhóm |
| 3 | `GROUP BY` | Nhóm các hàng |
| 4 | `HAVING` | Lọc nhóm sau khi nhóm |
| 5 | `SELECT` | Chọn cột hiển thị |
| 6 | `DISTINCT` | Loại bỏ trùng lặp |
| 7 | `ORDER BY` | Sắp xếp kết quả |
| 8 | `LIMIT` / `OFFSET` | Giới hạn số hàng |

> __Lưu ý__: Vì `WHERE` thực thi trước `SELECT`, bạn __không thể__ dùng alias trong `WHERE`. Nhưng `ORDER BY` thực thi sau `SELECT`, nên __có thể__ dùng alias.

```sql
-- ❌ SAI: alias không dùng được trong WHERE
SELECT salary * 12 AS annual_salary FROM employees
WHERE annual_salary > 100000000;

-- ✅ ĐÚNG: dùng alias trong ORDER BY
SELECT salary * 12 AS annual_salary FROM employees
ORDER BY annual_salary DESC;
```

## 2. WHERE — Lọc dữ liệu

### 2.1. Toán tử so sánh

| Toán tử | Mô tả | Ví dụ |
| ------- | ----- | ----- |
| `=` | Bằng | `WHERE age = 25` |
| `<>` hoặc `!=` | Khác | `WHERE status <> 'INACTIVE'` |
| `<`, `>`, `<=`, `>=` | So sánh | `WHERE salary >= 10000000` |
| `BETWEEN` | Trong khoảng | `WHERE salary BETWEEN 10000000 AND 20000000` |
| `IN` | Trong tập giá trị | `WHERE department IN ('IT', 'HR')` |
| `LIKE` | So khớp mẫu | `WHERE name LIKE 'Nguyen%'` |
| `IS NULL` | Là NULL | `WHERE email IS NULL` |
| `IS NOT NULL` | Không NULL | `WHERE phone IS NOT NULL` |

<br />

### 2.2. Wildcard với LIKE

| Wildcard | Mô tả | Ví dụ |
| -------- | ----- | ----- |
| `%` | Khớp __0 hoặc nhiều__ ký tự bất kỳ | `LIKE 'Nguyen%'` → Nguyen Van A, Nguyen Thi B |
| `_` | Khớp __đúng 1__ ký tự | `LIKE '_an'` → Van, Tan |

```sql
-- Tìm tên bắt đầu bằng "Nguyen"
SELECT * FROM employees WHERE name LIKE 'Nguyen%';

-- Tìm email chứa "gmail"
SELECT * FROM employees WHERE email LIKE '%gmail%';

-- Tìm tên có đúng 5 ký tự
SELECT * FROM employees WHERE name LIKE '_____';
```

<br />

### 2.3. WHERE vs HAVING

| Tiêu chí | WHERE | HAVING |
| -------- | ----- | ------ |
| __Thời điểm__ | Lọc __trước__ GROUP BY | Lọc __sau__ GROUP BY |
| __Áp dụng cho__ | Từng hàng riêng lẻ | Nhóm đã được gom |
| __Hàm aggregate__ | ❌ Không sử dụng được | ✅ Sử dụng được |

```sql
-- WHERE: Lọc trước khi nhóm
SELECT department, AVG(salary) AS avg_salary
FROM employees
WHERE status = 'ACTIVE'        -- Lọc nhân viên active trước
GROUP BY department;

-- HAVING: Lọc sau khi nhóm
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 15000000;  -- Lọc phòng ban có lương TB > 15 triệu

-- Kết hợp WHERE và HAVING
SELECT department, COUNT(*) AS total, AVG(salary) AS avg_salary
FROM employees
WHERE status = 'ACTIVE'
GROUP BY department
HAVING COUNT(*) > 5
ORDER BY avg_salary DESC;
```

## 3. JOIN — Kết hợp bảng

### 3.1. Các loại JOIN

| Loại | Mô tả | Kết quả |
| ---- | ----- | ------- |
| __INNER JOIN__ | Chỉ lấy hàng __khớp ở cả 2 bảng__ | Giao của A và B |
| __LEFT JOIN__ | Tất cả hàng bảng trái + hàng khớp bảng phải | A ∪ (A ∩ B) |
| __RIGHT JOIN__ | Tất cả hàng bảng phải + hàng khớp bảng trái | B ∪ (A ∩ B) |
| __FULL OUTER JOIN__ | Tất cả hàng từ __cả 2 bảng__ | A ∪ B |
| __CROSS JOIN__ | Tích Descartes — mọi kết hợp có thể | m × n hàng |
| __SELF JOIN__ | Bảng join với chính nó | — |

<br />

### 3.2. Ví dụ JOIN

Giả sử có 2 bảng:

__employees__
| id | name | department_id |
| -- | ---- | ------------- |
| 1 | An | 1 |
| 2 | Binh | 2 |
| 3 | Cuong | NULL |

__departments__
| id | name |
| -- | ---- |
| 1 | Engineering |
| 2 | Marketing |
| 3 | HR |

```sql
-- INNER JOIN: Chỉ nhân viên CÓ phòng ban
SELECT e.name, d.name AS department
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;
-- Kết quả: An-Engineering, Binh-Marketing (Cuong không hiện vì NULL)

-- LEFT JOIN: Tất cả nhân viên, kể cả chưa có phòng ban
SELECT e.name, d.name AS department
FROM employees e
LEFT JOIN departments d ON e.department_id = d.id;
-- Kết quả: An-Engineering, Binh-Marketing, Cuong-NULL

-- RIGHT JOIN: Tất cả phòng ban, kể cả không có nhân viên
SELECT e.name, d.name AS department
FROM employees e
RIGHT JOIN departments d ON e.department_id = d.id;
-- Kết quả: An-Engineering, Binh-Marketing, NULL-HR

-- SELF JOIN: Tìm quản lý của nhân viên
SELECT e.name AS employee, m.name AS manager
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;
```

<br />

### 3.3. CROSS JOIN

__CROSS JOIN__ tạo __tích Descartes__ — kết hợp mỗi hàng bảng A với __mọi hàng__ bảng B.

```sql
-- Bảng sizes: S, M, L
-- Bảng colors: Red, Blue
-- CROSS JOIN → 6 kết hợp: S-Red, S-Blue, M-Red, M-Blue, L-Red, L-Blue
SELECT s.size, c.color
FROM sizes s
CROSS JOIN colors c;
```

> __Cẩn thận__: CROSS JOIN có thể tạo __rất nhiều hàng__ (m × n). Chỉ dùng khi thực sự cần.

## 4. NULL — Giá trị đặc biệt

### 4.1. NULL là gì?

__NULL__ đại diện cho giá trị __không xác định__ hoặc __không tồn tại__. NULL __không phải__ là `0`, chuỗi rỗng `""`, hay `false`.

<br />

### 4.2. Xử lý NULL

```sql
-- Kiểm tra NULL
SELECT * FROM employees WHERE phone IS NULL;
SELECT * FROM employees WHERE phone IS NOT NULL;

-- ❌ SAI: Không dùng = với NULL
SELECT * FROM employees WHERE phone = NULL;  -- LUÔN trả về rỗng!

-- Thay thế NULL bằng giá trị mặc định
SELECT name, COALESCE(phone, 'Chưa cập nhật') AS phone FROM employees;

-- IFNULL (MySQL)
SELECT name, IFNULL(phone, 'N/A') AS phone FROM employees;

-- CASE WHEN
SELECT name,
    CASE WHEN phone IS NULL THEN 'Chưa cập nhật'
         ELSE phone
    END AS phone
FROM employees;
```

<br />

### 4.3. NULL trong tính toán

```sql
-- Bất kỳ phép tính nào với NULL đều cho NULL
SELECT 5 + NULL;      -- NULL
SELECT 'abc' || NULL;  -- NULL
SELECT NULL = NULL;    -- NULL (không phải TRUE!)
SELECT NULL <> NULL;   -- NULL (không phải TRUE!)
```

## 5. Aggregate Functions — Hàm tổng hợp

### 5.1. Các hàm phổ biến

| Hàm | Mô tả | Ví dụ |
| --- | ----- | ----- |
| `COUNT()` | Đếm số hàng | `COUNT(*)`, `COUNT(column)` |
| `SUM()` | Tổng | `SUM(salary)` |
| `AVG()` | Trung bình | `AVG(salary)` |
| `MAX()` | Giá trị lớn nhất | `MAX(salary)` |
| `MIN()` | Giá trị nhỏ nhất | `MIN(salary)` |

> __Lưu ý__: `COUNT(*)` đếm __tất cả hàng__ (kể cả NULL). `COUNT(column)` đếm hàng __không NULL__.

```sql
-- Thống kê theo phòng ban
SELECT
    department,
    COUNT(*) AS total_employees,
    AVG(salary) AS avg_salary,
    MAX(salary) AS max_salary,
    MIN(salary) AS min_salary,
    SUM(salary) AS total_salary
FROM employees
GROUP BY department
ORDER BY avg_salary DESC;
```

## 6. Subquery — Truy vấn con

### 6.1. Subquery là gì?

__Subquery__ (truy vấn con) là một câu lệnh `SELECT` nằm bên trong một câu lệnh SQL khác.

```sql
-- Subquery trong WHERE: Tìm nhân viên có lương cao hơn trung bình
SELECT name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- Subquery trong FROM: Dùng kết quả subquery như 1 bảng tạm
SELECT dept_stats.department, dept_stats.avg_salary
FROM (
    SELECT department, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department
) AS dept_stats
WHERE dept_stats.avg_salary > 15000000;

-- Subquery trong SELECT
SELECT name, salary,
    (SELECT AVG(salary) FROM employees) AS company_avg,
    salary - (SELECT AVG(salary) FROM employees) AS diff_from_avg
FROM employees;
```

<br />

### 6.2. Correlated Subquery

__Correlated Subquery__ (truy vấn con tương quan) là subquery tham chiếu đến cột của outer query. Nó __chạy lại cho mỗi hàng__ của outer query.

```sql
-- Tìm nhân viên có lương cao nhất trong phòng ban của họ
SELECT e.name, e.salary, e.department
FROM employees e
WHERE e.salary = (
    SELECT MAX(e2.salary)
    FROM employees e2
    WHERE e2.department = e.department  -- Tham chiếu outer query
);
```

<br />

### 6.3. Subquery vs JOIN

| Tiêu chí | Subquery | JOIN |
| -------- | -------- | ---- |
| __Đọc hiểu__ | Dễ hiểu cho query đơn giản | Phức tạp hơn nhưng mạnh mẽ |
| __Hiệu suất__ | Có thể chậm (chạy nhiều lần) | Thường nhanh hơn (optimized) |
| __Kết quả__ | Trả về 1 giá trị hoặc tập giá trị | Trả về nhiều cột từ nhiều bảng |
| __Khi nào dùng__ | Lọc, so sánh với aggregate | Kết hợp dữ liệu từ nhiều bảng |

> __Best Practice__: Ưu tiên dùng __JOIN__ khi có thể. Dùng __Subquery__ khi cần so sánh với kết quả aggregate hoặc logic phức tạp.

## 7. Set Operators — Phép toán tập hợp

### 7.1. UNION, INTERSECT, EXCEPT

| Phép toán | Mô tả |
| --------- | ----- |
| `UNION` | Hợp 2 tập kết quả, __loại bỏ__ trùng lặp |
| `UNION ALL` | Hợp 2 tập kết quả, __giữ__ trùng lặp |
| `INTERSECT` | Giao — chỉ lấy hàng __có mặt ở cả 2__ |
| `EXCEPT` | Hiệu — lấy hàng __có ở tập 1 nhưng không có ở tập 2__ |

```sql
-- UNION: Danh sách tất cả người (nhân viên + khách hàng), loại trùng
SELECT name, email FROM employees
UNION
SELECT name, email FROM customers;

-- UNION ALL: Giữ trùng lặp (nhanh hơn)
SELECT name FROM employees
UNION ALL
SELECT name FROM customers;

-- INTERSECT: Người vừa là nhân viên vừa là khách hàng
SELECT email FROM employees
INTERSECT
SELECT email FROM customers;

-- EXCEPT: Nhân viên KHÔNG phải là khách hàng
SELECT email FROM employees
EXCEPT
SELECT email FROM customers;
```

> __Quy tắc__: Các câu SELECT phải có __cùng số cột__ và __kiểu dữ liệu tương thích__.

## 8. DELETE vs TRUNCATE vs DROP

| Tiêu chí | DELETE | TRUNCATE | DROP |
| -------- | ------ | -------- | ---- |
| __Loại__ | DML | DDL | DDL |
| __Tác dụng__ | Xóa __một số hoặc tất cả__ hàng | Xóa __tất cả__ hàng | Xóa __cả bảng__ (cấu trúc + dữ liệu) |
| __WHERE__ | ✅ Hỗ trợ | ❌ Không | ❌ Không |
| __Rollback__ | ✅ Có thể (nếu trong transaction) | ❌ Không thể | ❌ Không thể |
| __Trigger__ | ✅ Kích hoạt | ❌ Không | ❌ Không |
| __Tốc độ__ | Chậm (xóa từng hàng) | Nhanh (xóa toàn bộ) | Nhanh nhất |
| __Auto-increment__ | Giữ nguyên | Reset về 0 | — |

```sql
-- DELETE: Xóa nhân viên cụ thể
DELETE FROM employees WHERE id = 5;

-- TRUNCATE: Xóa hết dữ liệu, giữ bảng
TRUNCATE TABLE employees;

-- DROP: Xóa hoàn toàn bảng
DROP TABLE employees;
```

## 9. Câu hỏi phỏng vấn thường gặp

__Q: JOIN và Subquery khác nhau thế nào? Khi nào dùng cái nào?__

JOIN kết hợp dữ liệu từ nhiều bảng theo hàng ngang. Subquery trả về 1 tập kết quả dùng trong WHERE/FROM/SELECT. JOIN thường nhanh hơn do được database optimizer tối ưu. Dùng Subquery khi cần so sánh với aggregate.

__Q: UNION khác UNION ALL thế nào?__

UNION loại bỏ duplicate (tốn thêm bước sort + dedup → chậm hơn). UNION ALL giữ duplicate (nhanh hơn). Dùng UNION ALL khi biết chắc không có trùng hoặc không cần loại trùng.

__Q: DELETE, TRUNCATE, DROP khác nhau thế nào?__

DELETE xóa từng hàng (có WHERE, có rollback). TRUNCATE xóa toàn bộ dữ liệu (không rollback, reset auto-increment). DROP xóa cả bảng lẫn cấu trúc.
