# Spring Boot Web (MVC và API)

<br />

## 1. Class Trung Tâm (Main Application)

```java
@SpringBootApplication
public class Fap16MvcApplication {
    public static void main(String[] args) {
        SpringApplication.run(Fap16MvcApplication.class, args);
    }
}
```
Annotation `@SpringBootApplication` là combo gộp 3 annotation khác:
1. `@Configuration`: Cấu hình app.
2. `@ComponentScan`: Quét các beans (`@Service`, `@Controller`...).
3. `@EnableAutoConfiguration`: Tự động khởi động Tomcat ngầm, cấu hình JPA/Hibernate tự động dựa trên thư viện mình thêm vào POM.

## 2. API vs Web Controller

Spring Boot cung cấp 2 mô hình xử lý Request qua URL:

### 2.1. REST API (Trả về JSON dữ liệu thô)
Dành cho ứng dụng chia tách Backend/Frontend (Mobile App, ReactJS). Dữ liệu được trả về dạng JSON để Client tự vẽ giao diện.

```java
@RestController // @Controller + @ResponseBody (Trực tiếp trả về dữ liệu Json)
@RequestMapping("/api/v1") // Tiền tố URL
public class CoffeeApiController {

    // Trả về JSON (Lấy qua URL GET)
    @GetMapping("/cup") 
    public Coffee getACup(){
        return new Coffee("JCNT", "Cafe Java", 36000); 
        // Spring dùng Jackson library ngầm convert Object Coffee thành Json string
    }

    // Nhận JSON từ FE gửi lên, lưu / chỉnh sửa và trả về lại
    @PostMapping("/add")
    public Coffee addACup(@RequestBody Coffee coffee){ // @RequestBody giúp hứng cục Json chuyển ngược thành Java Object
        coffee.setName(coffee.getName() + " | Ngon");
        return coffee; // Return 201 created (thường cấu hình ở Http status)
    }
}
```

### 2.2. MVC Web (Trả về trang HTML hoàn chỉnh)
Dùng Server-Side Rendering (Thymeleaf, JSP). Controller chuẩn bị dữ liệu (Model) và nhét vào cái thùng chứa, ném thùng chứa đó cho trang HTML (View) "múc" data trộn lên rồi trả file HTML về trình duyệt.

```java
@Controller // Chuyên trả về tên một trang Web / View (HTML)
public class ProductController {

    // Khi người dùng gõ localhost/products
    @RequestMapping(path = "/products", method = RequestMethod.GET)
    public String list(Model model){ // Thùng chứa model do Spring bơm vào tự động
        
        // 1. Chuẩn bị data (Có thể gọi Service/Repo để lấy từ DB)
        List<Product> products = new ArrayList<>();
        products.add(new Product("1", "Iphone", 30000000));
        
        // 2. Bỏ data vào thùng chứa Model: model.addAttribute("tên_nhãn", giá_trị)
        model.addAttribute("msg", "Xin chào admin");
        model.addAttribute("products", products);
        
        // 3. Trả về VIEW. Thymeleaf sẽ tìm file "products.html" trong thư mục resources/templates
        return "products"; 
    }
}
```
**Bên View (`products.html`):** Sử dụng cú pháp Thymeleaf (`${tên_nhãn}`) để móc cục data trong thùng chứa model ra hiển thị.

## 3. Hiện tượng Resubmission Form (Lỗi F5 trang Post)

### 3.1. Vấn đề của POST Request
Khi bạn điền Form (Thêm mới sinh viên, sửa sản phẩm) và bấm Submit, data được đẩy lên server qua URL với HTTP method là `POST`.
Nếu Controller xử lý xong và return thẳng tên trang (ví dụ `return "result_page"`), url trình duyệt vẫn giữ nguyên đường dẫn của hàm POST cũ.
- Nếu User bất cẩn **Bấm F5 (Refresh) trang kết quả**, trình duyệt sẽ hiểu là: *"Hãy thực hiện lại lệnh POST cũ"*.
- Hậu quả: Dữ liệu được **Submit lại lần nữa (Duplicate Data)**, có nguy cơ thêm 2 dòng trùng lặp vào CSDL!

### 3.2. Giải pháp: Redirect (PRG Pattern: Post-Redirect-Get)
Xử lý POST xong thì **KHÔNG ĐƯỢC return view trực tiếp**, mà phải yêu cầu trình duyệt ĐỔI ĐƯỜNG DẪN sang một URL mới (URL Get) bằng từ khóa `redirect:`.

```java
@PostMapping("/products/update")
public String updateProduct(Product p, RedirectAttributes redirectAtts) {
    // 1. Xử lý update vào DB
    productService.update(p);

    // 2. Thay vì dùng Model (chỉ sống trong 1 request), phải dùng RedirectAttributes để gửi data xuyên qua đợt redirect
    redirectAtts.addFlashAttribute("msg", "Update thành công!");
    
    // 3. Redirect sang URL GET của trang xem kết quả
    return "redirect:/products"; // Trình duyệt bị đá văng sang URL mới, F5 không bị duplicate form nữa!
}
```
