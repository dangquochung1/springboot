# Maven và Quản lý Dependencies

<br />

## 1. Maven là gì?

**Maven** là một công cụ quản lý dự án và quản lý thư viện (dependencies) cực kỳ phổ biến trong Java. Thay vì phải tải các file `.jar` thủ công và add vào project, Maven sẽ tự động tải các thư viện này từ kho lưu trữ trung tâm (Maven Central) về máy của bạn.

Mọi thứ trong Maven được cấu hình qua file `pom.xml` (Project Object Model).

<br />

## 2. Các lệnh Maven cơ bản

Bạn có thể chạy các lệnh Maven qua terminal (nhớ cài Maven trước, hoặc dùng `./mvnw` đi kèm trong Spring Boot project):

| Lệnh | Chức năng |
| ---- | -------- |
| `mvn clean` | Xóa thư mục `target/` chứa các file đã build cũ |
| `mvn compile` | Biên dịch source code (từ `.java` sang `.class`) |
| `mvn test` | Chạy các unit tests |
| `mvn package` | Đóng gói project thành file `.jar` hoặc `.war` (đặt trong thư mục `target`) |
| `mvn install` | Đóng gói và "cài đặt" file `.jar` vào kho Maven local (thường là `~/.m2/repository`), để các project khác trên cùng máy có thể dùng lại |

> **Mẹo**: Thường chúng ta hay chạy kết hợp `mvn clean package` hoặc `mvn clean install` để đảm bảo build từ đầu một cách sạch sẽ.

<br />

## 3. Đóng gói và chạy file JAR

Khi dùng `mvn clean package`, Maven sẽ tạo ra một file `.jar` trong thư mục `target`.

- **Chạy thông thường (có chứa class Main)**:
  ```bash
  java -jar .\target\ten-app-1.0.jar
  ```
  _Lưu ý_: Nếu có nhiều class Main, Maven có thể không biết gọi class nào trừ khi được chỉ định rõ trong cấu hình manifest.
  
- **Chạy khi chỉ định class Main cụ thể (chưa cấu hình manifest)**:
  ```bash
  java -cp .\target\ten-app-1.0.jar com.hungdang.basispractise.Main
  ```

> **Kinh nghiệm debug**: Để kiểm tra file `.jar` có chứa hàm `main` được cấu hình đúng không, bạn có thể đổi đuôi `.jar` thành `.rar` hoặc `.zip`, giải nén ra và kiểm tra thư mục `META-INF/MANIFEST.MF`.

<br />

## 4. Quản lý Package (Tên miền đảo ngược)

Trong Java và Maven, groupId (tên package gốc) thường được đặt theo quy tắc **tên miền đảo ngược** (reverse domain name).

Ví dụ: `com.danghung`, `org.springframework`
- Giúp gom nhóm các dự án có chung nguồn gốc.
- Tránh trùng lặp tên class giữa các công ty/cá nhân khác nhau trên toàn thế giới.

<br />

## 5. Lưu ý khi làm việc với Multi-module (Dependency nội bộ)

Giả sử bạn có project A (chứa thư viện tiện ích `maventogo`) và project B (ứng dụng dùng thư viện `maventogo`).
- Nếu bạn chỉ code bằng IntelliJ, nó có thể tự link project A và B với nhau.
- Nhưng khi build bằng dòng lệnh hoặc đem sang máy khác chạy bằng lệnh `java -cp...`, nó sẽ báo lỗi **"không tìm thấy thư viện"**.
- **Giải pháp**: 
  - Tại project A, bạn phải chạy `mvn clean install` (để tống file JAR vào kho `.m2` local).
  - Hoặc trong Spring Boot plugin, cấu hình để đóng gói luôn các thư viện nội bộ vào chung một file "fat JAR".
  - **Lưu ý**: Trong project thư viện (project A), nên xóa hàm `main` đi để tránh việc Maven nhầm lẫn cấu hình đóng gói.
