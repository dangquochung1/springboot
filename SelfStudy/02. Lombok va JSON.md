# Lombok và JSON

<br />

## 1. Boilerplate Code và Lombok

### 1.1. Nỗi khổ Boilerplate Code
Trong Java, khi tạo một class (ví dụ Entity hay DTO), chúng ta bắt buộc phải viết rất nhiều code mang tính lặp đi lặp lại một cách nhàm chán (boilerplate code), bao gồm:
1. Constructor rỗng (Empty Constructor / No-Args Constructor).
2. Constructor đầy đủ tham số (All-Args Constructor).
3. Getter/Setter cho tất cả các thuộc tính.
4. Hàm `toString()` để in thông tin object ra console dễ nhìn.

Ví dụ, một class `Student` có 4 thuộc tính nhưng có thể mất đến hàng chục dòng code chỉ cho getter/setter.

<br />

### 1.2. Giải pháp: Lombok
**Lombok** là một thư viện của bên thứ ba ("hàng anh em mạng", không phải của JDK chính hãng như C# có sẵn properties) giúp tự động sinh ra các đoạn boilerplate code này tại thời điểm biên dịch (compile time) chỉ bằng các **Annotations**.

Các Annotations thường dùng:
- `@NoArgsConstructor`: Tạo constructor rỗng.
- `@AllArgsConstructor`: Tạo constructor full tham số.
- `@Getter` / `@Setter`: Tạo các hàm getter/setter.
- `@ToString`: Tạo hàm toString.
- `@Data`: Gộp chung của `@Getter`, `@Setter`, `@ToString`, `@EqualsAndHashCode`, và `@RequiredArgsConstructor`.
- `@Builder`: Cung cấp Builder Pattern giúp khởi tạo object dễ dàng hơn mà không cần nhớ thứ tự tham số trong constructor.

**Ví dụ class gọn gàng nhờ Lombok:**
```java
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Student {
    private String id;
    private String name;
    private int yob;
    private double gpa;
}

// Khởi tạo siêu tiện với Builder
Student s = Student.builder()
                   .id("SE123")
                   .name("Nguyen Van A")
                   .yob(2000)
                   .build();
```

## 2. JSON — Tiêu chuẩn giao tiếp dữ liệu

### 2.1. JSON là gì?
**JSON (JavaScript Object Notation)** là kỹ thuật biểu diễn thông tin của một đối tượng (object) dưới dạng văn bản (text), dựa trên cú pháp của ngôn ngữ JavaScript.

So sánh việc tạo object:
- **Java**: `new Student("Nguyen Van A", 20, "Ha Noi");` 
  *(Nhìn vào text không tự giải thích được số 20 là gì, phải đoán)*
- **JSON**:
  ```json
  {
      "name": "Nguyen Van A",
      "age": 20,
      "address": "Ha Noi"
  }
  ```
  *(Rất self-explanatory, mô tả rõ ràng key-value)*

### 2.2. Vai trò của JSON trong Web API
Ngày nay, mô hình phát triển phần mềm thường chia làm 2 cục riêng biệt: **Frontend (FE)** và **Backend (BE)**.
- Trong code BE (Java, C#), dữ liệu tồn tại dưới dạng **Object**.
- Khi truyền dữ liệu qua lại trên mạng (giao tiếp HTTP), Object không thể "bay" qua dây mạng được. Dữ liệu cần được chuyển thành dạng text (JSON).
- FE (Javascript, React, Vue dùng Axios) nhận JSON và hiển thị lên màn hình.

**Flow truyền dữ liệu**: `Java Object  <-- Convert -->  JSON text  <-- Internet -->  FE Client`

### 2.3. Các thư viện Convert (Serialize/Deserialize)
Để convert tự động qua lại giữa Object và JSON, chúng ta dùng các thư viện:
- **Java**: Jackson (mặc định trong Spring Boot), Gson (của Google).
- **C#**: Newtonsoft.Json.

### 2.4. JSON Array
Nếu chỉ có 1 object thì dùng dấu `{ }`. 
Nếu trả về một danh sách (List/Array) thì JSON sẽ bọc trong dấu `[ ]`.

```json
[
  {"gpa":3.5, "id":"HE123", "name":"Hun Ci", "yob":2000},
  {"gpa":3.8, "id":"HE124", "name":"Mai An", "yob":2001}
]
```

> **Lưu ý xử lý lỗi**: Khi convert hoặc "truyền banh" (xử lý luồng data), rất dễ xảy ra Exception (ví dụ: chuỗi JSON sai format). Nguyên tắc là phải có `try-catch` hoặc `throws` Exception để code không bị sập ngang.
