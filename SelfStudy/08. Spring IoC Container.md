# Spring IoC Container

<br />

## 1. IoC Container là gì?
Ở bài trước (SOLID & DI), để giải quyết Tight Coupling, chúng ta phải "Tiêm bằng tay" (Manual DI): tự `new` Dependency (`WhatsAppSender`) ở class Main rồi nhét nó qua Constructor cho class chính (`UserService`).

Nhưng nếu dự án có hàng nghìn class thì sao? Ai sẽ tự đi `new` hàng nghìn thứ và móc nối chúng lại với nhau theo đúng thứ tự?

**IoC (Inversion of Control - Đảo ngược điều khiển)**: Bạn sẽ giao việc "khởi tạo Object (new) và bơm/tiêm chúng vào nhau" cho một thằng quản gia lo liệu. Thằng quản gia đó trong framework Spring gọi là **Spring IoC Container** (hay ApplicationContext).

> "Em kiểm soát những đứa phụ thuộc, em chích/tiêm chúng vào cho anh. Anh chỉ việc xài thôi!"

<br />

## 2. Biến Object thành Bean (Đánh dấu cho Container)

Muốn Container lo việc tạo object giùm mình, mình phải "đánh dấu" các class đó bằng Annotations. Object do Spring Container tạo và quản lý được gọi là **Bean**.

- `@Component`: Đánh dấu một class chung chung cần Spring tạo ra giùm.
- `@Service`: Dùng cho class xử lý nghiệp vụ (Business Logic).
- `@Repository`: Dùng cho class tương tác Database.
- `@Controller` / `@RestController`: Dùng cho class xử lý request từ web/URL.

```java
@Component // Báo cho Spring: Hãy tự "new" tôi đi!
public class PdfGenerator {
    public void generateFile(String name) { ... }
}
```

## 3. Các cách Tiêm (Injection) trong Spring

Có 3 cách để báo cho Spring biết: "Ê Spring, hãy nhặt cái Bean (vd: PdfGenerator) mài đang giữ, tiêm vào chỗ này cho tao xài". Chúng ta sử dụng annotation `@Autowired`.

### Cách 1: Field Injection
Đây là cách ngắn gọn nhất, Spring dùng cơ chế Reflection để tiêm trực tiếp vào biến bộ nhớ lúc Runtime.
```java
@Service("ngocTrinh") // Đặt tên Bean cụ thể
public class ContractService {
    
    @Autowired // Cứ thế tiêm vào Field
    private PdfGenerator gen;

    public void processContract() {
        gen.generateFile("test.pdf"); 
    }
}
```
**Nhược điểm:** Khó viết Unit Test do phải phụ thuộc vào Spring khởi động để tiêm dependency.

### Cách 2: Setter Injection
Tiêm thông qua một hàm Setter. Ít dùng, mang ý nghĩa thay đổi / sửa lại dependency sau này (nếu có).
```java
@Service
public class ContractService {
    private PdfGenerator gen;

    @Autowired // Bắt buộc có Autowired ở đây
    public void setPdfGenerator(PdfGenerator gen) {
        this.gen = gen;
    }
}
```

### Cách 3: Constructor Injection (Được khuyên dùng nhất)
Là cách tốt nhất và được giang hồ ưa chuộng nhất. Injection diễn ra ngay khi object được tạo nên đảm bảo an toàn tuyệt đối, tránh hiện tượng NullPointerException.

```java
@Service
public class ContractService {
    private PdfGenerator gen;

    @Autowired // Tiêm qua constructor. (Lưu ý: Nếu chỉ có duy nhất 1 constructor thì có thể bỏ @Autowired, Spring tự hiểu).
    public ContractService(PdfGenerator gen) {
        this.gen = gen;
    }
}
```

## 4. Tổng kết quá trình hoạt động của Spring Application
1. **Quét (Component Scan)**: Khi chạy ứng dụng, Spring sẽ quét toàn bộ package (và các package con) để tìm những class gắn mác `@Component`, `@Service`...
2. **Khởi tạo (Instantiate)**: Tạo một (và chỉ một - theo kiểu Singleton) bản sao vào trong bộ nhớ (IoC Container).
3. **Tiêm (Inject)**: Giải quyết các sự phụ thuộc (`@Autowired`), lấy Bean này đút vào Bean kia một cách tự động.

Toàn bộ quá trình DI bằng tay (Manual) giờ đây được Spring thực hiện hoàn toàn tàng hình, dev chỉ tập trung vào viết business logic!
