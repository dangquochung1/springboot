# JDBC — Kết nối Database Thuần

<br />

## 1. Giới thiệu JDBC
**JDBC (Java Database Connectivity)** là bộ API chuẩn của Java cung cấp các interface và class để ứng dụng Java có thể kết nối và thực thi các câu lệnh SQL với Cơ sở dữ liệu quan hệ (SQL Server, MySQL, Oracle, v.v.).

Đây là cách kết nối ở tầng **"thấp" nhất** (native nhất). Các Framework hiện đại như Hibernate, JPA, Spring Data JPA bản chất bên dưới vẫn sử dụng JDBC để tương tác với Database.

<br />

## 2. Các bước kết nối Database với JDBC

### Bước 1: Chuẩn bị thông tin kết nối (Connection String)
Để kết nối, bạn cần URL của DB, Username và Password.

```java
String dbURL = "jdbc:sqlserver://localhost:1433;databaseName=hsf302;encrypt=true;trustServerCertificate=true";
String user = "sa";
String pass = "123";
```

### Bước 2: Tạo đối tượng `Connection`
Sử dụng `DriverManager` để thiết lập kết nối tới Database.

```java
Connection conn = null;
conn = DriverManager.getConnection(dbURL, user, pass);
System.out.println("Connect to DB successfully");
```

### Bước 3: Tạo `PreparedStatement` và viết câu lệnh SQL
`PreparedStatement` giúp bạn định nghĩa câu query SQL cần chạy (có hỗ trợ truyền tham số để chống SQL Injection).

```java
PreparedStatement ps = conn.prepareStatement("select * from subject");
```

### Bước 4: Thực thi và lấy `ResultSet`
Gọi hàm `executeQuery()` (đối với lệnh SELECT). Kết quả trả về là một `ResultSet`. `ResultSet` giống như một ArrayList chứa nhiều bản ghi (record) từ bảng trong Database.

```java
ResultSet rs = ps.executeQuery();
```

### Bước 5: Duyệt `ResultSet` (Loop)
Dùng vòng lặp `while (rs.next())` để duyệt từng dòng dữ liệu. 
**Nhược điểm cực lớn của JDBC thuần**: Bạn phải nhớ chính xác *tên cột* và *kiểu dữ liệu* (data type) của từng cột trong database để dùng hàm get tương ứng (`getString`, `getInt`, v.v.).

```java
while (rs.next()) {
    String code = rs.getString("code");
    String name = rs.getString("name");
    int credit = rs.getInt("credits");
    int hours = rs.getInt("studyhours");

    // In ra màn hình (sử dụng printf để căn gióng cho đẹp)
    System.out.printf("|%10s|%-40s|%2d|%4d|\n", code, name, credit, hours);
}
```

### Bước 6: Đóng kết nối
Luôn luôn đóng `Connection` sau khi sử dụng xong để giải phóng tài nguyên.
```java
conn.close();
```

## 3. Code Full Ví dụ

```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;

public class Main {
    public static void main(String[] args) {
        try {
            String dbURL = "jdbc:sqlserver://localhost:1433;databaseName=hsf302;encrypt=true;trustServerCertificate=true";
            String user = "sa";
            String pass = "123";
            
            Connection conn = DriverManager.getConnection(dbURL, user, pass);
            System.out.println("Connect to DB successfully");

            PreparedStatement ps = conn.prepareStatement("select * from subject");
            ResultSet rs = ps.executeQuery();
            
            while (rs.next()) {
               String code = rs.getString("code");
               String name = rs.getString("name");
               int credit = rs.getInt("credits");
               int hours = rs.getInt("studyhours");
               
               System.out.printf("|%10s|%-40s|%2d|%4d|\n", code, name, credit, hours);
            }
            conn.close();
        } catch (Exception ex) {
            ex.printStackTrace();
        }
    }
}
```

## 4. Tại sao người ta ít dùng JDBC thuần ngày nay?
Như đã thấy trong mã code:
- Phải tự map dữ liệu từ cột SQL (`rs.getString("code")`) sang biến Java. Rất thủ công và dễ sai sót chính tả.
- Code bị lặp lại rất nhiều (mở connection, đóng connection, try-catch).
- Không theo tư duy Hướng đối tượng (OOP).

**Giải pháp**: Sử dụng **ORM (Object-Relational Mapping)** như JPA/Hibernate để ánh xạ trực tiếp một class Java thành một bảng trong CSDL (sẽ học ở bài sau).
