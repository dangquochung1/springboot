# ORM, JPA và JPQL

<br />

## 1. ORM và JPA là gì?

- **ORM (Object-Relational Mapping)**: Là kỹ thuật lập trình giúp ánh xạ trực tiếp các Class trong thế giới Hướng Đối Tượng (OOP - Java) xuống các Table trong thế giới Database Relational (SQL). Nhờ ORM, developer không cần viết SQL thủ công (như bài JDBC) mà có thể dùng thẳng Object để thao tác với DB.
- **JPA (Java Persistence API)**: Là một bộ tiêu chuẩn (interface) của Java định nghĩa cách thực hiện ORM. 
- **Hibernate**: Là một "nhà thầu" (implementation) nổi tiếng nhất triển khai bộ tiêu chuẩn JPA (nhà thầu khác có thể là EclipseLink). JDBC được Hibernate dùng ngầm bên dưới.

## 2. Các Annotation ánh xạ (@Entity)

Để biến một class thành một table, ta dùng các JPA Annotations:

```java
import jakarta.persistence.*;

@Entity
@Table(name = "Student") // Tùy chọn, nếu không có mặc định lấy tên class = tên table
public class Student {

    @Id
    @Column(name = "Id", columnDefinition = "CHAR(8)") 
    private String id;  

    // @Nationalized (Của Hibernate) giúp lưu tiếng việt có dấu (NVARCHAR)
    // Tuy nhiên dùng columnDefinition = "NVARCHAR(50)" sẽ giữ được tính khả chuyển nếu đổi nhà thầu JPA khác
    @Column(name = "Name", nullable = false, length = 50, columnDefinition = "NVARCHAR(50)")
    private String name;

    @Column(name = "Yob", nullable = false)
    private int yob;

    // Nếu không có @Column, mặc định lấy tên thuộc tính biến thành tên cột
    @Column(name = "GPA")
    private double gpa;

    // Bắt buộc phải có Constructor rỗng + Getter/Setter
    public Student() {}
}
```

## 3. EntityManager — "Ông sếp" quản lý Entity

`EntityManager` là nhân vật trung tâm trong JPA. Nó làm nhiệm vụ kết nối CSDL và cung cấp các hàm thực hiện CRUD tự động sinh ra SQL ngầm.

- `persist(object)`: Tương đương lệnh `INSERT`. Thêm mới.
- `find(Class, PK)`: Tương đương lệnh `SELECT * WHERE id = ...`. Tìm một dòng theo Primary Key.
- `merge(object)`: Tương đương lệnh `UPDATE`. (Nếu object chưa có thì có thể insert tùy cấu hình).
- `remove(object)`: Tương đương lệnh `DELETE`.

**Luôn làm việc trên 1 Record cụ thể qua PK:** 
Trong SWP hoặc dự án thực tế, người dùng xem danh sách, click vào nút "Sửa" / "Xóa" trên một dòng. Thao tác này truyền ID xuống DB, ta dùng `find(Class, ID)` để lôi object đó lên trước khi thao tác tiếp.

## 4. Transaction (Giao dịch)

Khi làm các hành động thay đổi hiện trạng database (Thêm, Sửa, Xóa), ta **bắt buộc** phải gói chúng trong một Transaction. 
Nguyên lý ACID: Do all or nothing (Hoặc tất cả thành công, hoặc không gì cả). (SELECT thì không cần transaction vì không thay đổi DB).

```java
EntityManagerFactory emf = Persistence.createEntityManagerFactory("my-pu");
EntityManager em = emf.createEntityManager();

// 1. Thêm mới (Persist)
em.getTransaction().begin();
Student an = new Student("SE123", "An", 2000, 8.5);
em.persist(an);
em.getTransaction().commit(); 

// 2. Cập nhật (Update)
em.getTransaction().begin();
Lecturer x = em.find(Lecturer.class, 2); // Kéo ông sếp ID=2 lên
x.setSalary(99_000_000); // Set lại giá trị
em.getTransaction().commit(); // Tự động update xuống DB

// 3. Xóa (Remove)
em.getTransaction().begin();
Student y = em.find(Student.class, "SE123");
em.remove(y);
em.getTransaction().commit();

em.close(); // Giải tán ông sếp khi xong việc
```

## 5. JPQL (Java Persistence Query Language)

Nếu `find()` chỉ lấy được 1 dòng theo ID, thì để lấy List danh sách hoặc filter (tìm kiếm linh hoạt), ta dùng **JPQL**.

- JPQL là ngôn ngữ "độ" lại từ SQL chuẩn, nhưng dành cho thế giới OOP. Thay vì truy vấn trên Table/Column, JPQL truy vấn trên **Class/Field**.

**SQL vs JPQL:**
- SQL: `SELECT * FROM lecturer WHERE salary = 20000000`
- JPQL: `SELECT x FROM Lecturer x WHERE x.salary = 20000000`
  - `Lecturer`: Là tên Java Class.
  - `x`: Là một biến đại diện cho từng Object (tương đương 1 dòng) sinh ra từ Class đó.
  - `x.salary`: Tên field trong Class.

**Truyền tham số (Parameter):**
```java
// Giống toán tử LIKE trong SQL
String jpql = "SELECT x FROM Lecturer x WHERE x.yob = :pYob AND x.name LIKE :pName";

List<Lecturer> result = em.createQuery(jpql, Lecturer.class)
                          .setParameter("pYob", 1996)
                          .setParameter("pName", "%Toan%")
                          .getResultList();
```

## 6. Functional Programming với Lambda

Để in danh sách kết quả, thay vì dùng vòng lặp `for` truyền thống, Java hỗ trợ cú pháp Lambda rất gọn nhẹ và chặt chẽ với Stream API:

```java
List<Student> result = em.createQuery("SELECT s FROM Student s", Student.class).getResultList();

System.out.println("Danh sách sinh viên:");
// Kiểu Lambda expression
result.forEach(stu -> {
    System.out.println(stu);
});

// Gọn hơn nữa bằng Method Reference:
result.forEach(System.out::println);
```
