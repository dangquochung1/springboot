# Layer Architecture và Singleton

<br />

## 1. Mẫu thiết kế Singleton Pattern

### 1.1. Vấn đề của EntityManagerFactory
Trong JPA, `EntityManagerFactory` là đối tượng chịu trách nhiệm đọc file cấu hình (`persistence.xml`), thiết lập kết nối (connection) tới SQL Server/MySQL. Nó vô cùng nặng nề (heavy-weight) và tốn tài nguyên RAM cũng như thời gian khởi tạo. 
- Nếu mỗi lần cần gọi database, ta lại dùng từ khóa `new` hay `createEntityManagerFactory()`, hệ thống sẽ bị giật lag và hao tổn bộ nhớ cực lớn. 
- Đối tượng này **chỉ nên được cấu hình/tạo ra 1 lần duy nhất** khi ứng dụng khởi động và tái sử dụng liên tục.

### 1.2. Giải pháp: Singleton
**Singleton** là thuật ngữ chỉ một thứ gì đó **chỉ có 1 phiên bản duy nhất** trong RAM. Ta thường dùng thiết kế này cho các class mang tính chất tiện ích (Utility). Trái ngược với class như `Student`, ta có thể `new` bao nhiêu sinh viên cũng được.

**Cách code Singleton ở Java Core (ví dụ JpaUtil):**
- Sử dụng kết hợp từ khóa `static` và `private constructor`.
- Nếu 1 class không có constructor, JVM tự tạo 1 cái rỗng (public). Do đó, ta **cố tình viết 1 private constructor rỗng** để "cấm" bên ngoài dùng lệnh `new`.
- Dùng khối `static { }` để đoạn code cấu hình được chạy trước tiên và chỉ chạy 1 lần.

```java
public class JpaUtil {
    
    // Biến static duy nhất, chứa "heavy load". Dùng final để không bị gán lại.
    private static final EntityManagerFactory emf;

    // Khối lệnh static, chạy đầu tiên và chỉ 1 lần khi class được load vào RAM
    static {
        try {
            // Load file cấu hình, tạo kết nối
            emf = Persistence.createEntityManagerFactory("com.danghung.superapp-PU");
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    // CẤM bên ngoài dùng lệnh "new JpaUtil()" để tránh sinh nhiều object
    private JpaUtil() {}

    // Method lấy EntityManager từ factory
    public static EntityManager getEntityManager() {
        return emf.createEntityManager();
    }

    // Đóng cổng kết nối khi tắt server
    public static void shutdown() {
        emf.close();
    }
}
```

> **Ghi chú Spring Boot**: Trong Spring Boot, bạn hầu như không phải viết tay class Singleton như thế này nữa, vì Spring IoC Container đã tự động biến tất cả các bean (`@Service`, `@Repository`, `EntityManager`) thành Singleton mặc định rồi!

## 2. Kiến trúc 3 Lớp (3-Layer Architecture)

Để code không bị rối rắm, gom thành một cục khổng lồ trong class Main, người ta chia tách code theo nhiệm vụ từng tầng (Layer).

**Luồng dữ liệu (Flow):** `UI (Controller) -> Service -> Repository -> JpaUtil (DB)`

<br />

### 2.1. Lớp Repository (Nhà kho dữ liệu)
- Tương tác trực tiếp với Database (bảng `student`).
- Phải nhờ vả ông `EntityManager` (Lấy từ `JpaUtil` Singleton) để thao tác.
- Tên hàm thường rất "kỹ thuật", giống các lệnh SQL chuẩn: `save()`, `update()`, `remove()`, `delete()`, `find()`, `findById()`, `findAll()`.

```java
public class StudentRepo {
    public void save(Student obj) {
        EntityManager em = JpaUtil.getEntityManager();
        em.getTransaction().begin();
        em.persist(obj);
        em.getTransaction().commit();
        em.close();
    }
    
    public List<Student> findAll() {
        EntityManager em = JpaUtil.getEntityManager();
        return em.createQuery("FROM Student ", Student.class).getResultList();
    }
}
```

<br />

### 2.2. Lớp Service (Xử lý nghiệp vụ)
- Đứng giữa Controller (Người dùng) và Repository. 
- Hứng thông tin từ người dùng, thực hiện validate (kiểm tra trùng lặp email, tính hợp lệ...), gọi các logic nghiệp vụ, sau đó mới đẩy cục data sạch xuống cho Repo lo việc ghi vào DB.
- Tên hàm thường mang tính "Nghiệp vụ" (gần với business/người dùng hơn): `createStudent()`, `getAllStudents()`, `updateStudent()`.

```java
public class StudentService {

    // Gọi đến tầng Repository
    private StudentRepo studentRepo = new StudentRepo();

    public void createStudent(Student obj) {
        // Todo logic: check trùng key, validate format số điện thoại, báo lỗi try-catch...
        // Nếu hợp lệ hết mới đẩy xuống DB
        studentRepo.save(obj);
    }

    public List<Student> getAllStudents() {
        return studentRepo.findAll();
    }
}
```

<br />

### 2.3. Lớp UI / Controller (Trong Core Java thường là hàm Main)
- Là nơi hiển thị giao diện, tiếp nhận thao tác của User, không chứa logic validate hay SQL. Nó chỉ "sai vặt" class Service.

```java
public class Main {
    public static void main(String[] args) {
        StudentService studentService = new StudentService();
        
        // Tạo mới sinh viên
        Student an = new Student("SE124555", "Tran Binh An", 2005, 8.5);
        studentService.createStudent(an);

        // Hiển thị danh sách
        System.out.println("Tất cả sinh viên:");
        studentService.getAllStudents().forEach(x -> System.out.println(x));
    }
}
```
