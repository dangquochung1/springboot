# Spring Bean và Bean Lifecycle

<br />

## 1. Spring Bean

### 1.1. Spring Bean là gì?

__Spring Bean__ là một đối tượng được __IoC Container__ (Spring Container) tạo ra, quản lý và lắp ráp. Bất kỳ class Java nào được đăng ký và quản lý bởi Spring Container đều được gọi là __Bean__.

Nói cách khác, Spring Bean __không phải__ là một class đặc biệt, mà là bất kỳ đối tượng nào mà Spring Container biết đến và quản lý lifecycle của nó.

<br />

### 1.2. Cách đăng ký Bean

Có 2 cách chính để đăng ký Bean trong Spring:

#### Cách 1: Sử dụng Stereotype Annotations

```java
@Component      // Bean chung
@Service        // Bean cho tầng Service (business logic)
@Repository     // Bean cho tầng Data Access
@Controller     // Bean cho tầng Controller (Spring MVC)
@RestController // Bean cho REST Controller (@Controller + @ResponseBody)
```

```java
@Service
public class UserService {
    // Spring tự động tạo Bean từ class này
}
```

#### Cách 2: Sử dụng `@Configuration` + `@Bean`

```java
@Configuration
public class AppConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public ModelMapper modelMapper() {
        return new ModelMapper();
    }
}
```

<br />

### 1.3. So sánh `@Component` vs `@Bean`

| Tiêu chí | `@Component` | `@Bean` |
| -------- | ------------ | ------- |
| __Đặt ở đâu__ | Trên __class__ | Trên __method__ (trong class `@Configuration`) |
| __Tạo Bean__ | Tự động qua __Component Scanning__ | Khai báo thủ công trong method |
| __Dùng khi__ | Class do __mình viết__ | Class từ __thư viện bên ngoài__ (không thể thêm annotation) |
| __Tùy chỉnh__ | Hạn chế | Linh hoạt (có thể customize logic khởi tạo) |

<br />

### 1.4. Stereotype Annotations

Các __Stereotype Annotations__ đều kế thừa từ `@Component`, nhưng mang ý nghĩa ngữ cảnh khác nhau:

| Annotation | Tầng (Layer) | Mục đích |
| ---------- | ------------ | -------- |
| `@Component` | Chung | Bean tổng quát, không thuộc tầng cụ thể |
| `@Service` | Service | Business logic, không thêm tính năng đặc biệt |
| `@Repository` | Data Access | Truy cập dữ liệu, tự động __exception translation__ (chuyển SQL exception → Spring DataAccessException) |
| `@Controller` | Presentation | Xử lý HTTP request, trả về __view name__ |
| `@RestController` | Presentation | `@Controller` + `@ResponseBody`, trả về __JSON/XML__ |

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    // @Repository giúp Spring tự động chuyển đổi
    // các database exception thành DataAccessException
}

@Service
public class UserService {
    // Business logic
}

@RestController
public class UserController {
    // REST API endpoints
}
```

## 2. Bean Scopes

### 2.1. Bean Scope là gì?

__Bean Scope__ xác định __phạm vi tồn tại__ và __số lượng instance__ của một Bean được tạo ra bởi Spring Container.

Spring hỗ trợ các scope sau:

| Scope | Mô tả | Số instance |
| ----- | ----- | ----------- |
| __singleton__ | Mặc định. Chỉ tạo __1 instance duy nhất__ trong toàn bộ IoC Container | 1 |
| __prototype__ | Tạo __instance mới__ mỗi khi được yêu cầu | N (mỗi lần request) |
| __request__ | Tạo 1 instance cho mỗi __HTTP request__ (chỉ dùng trong Web) | 1 per request |
| __session__ | Tạo 1 instance cho mỗi __HTTP session__ (chỉ dùng trong Web) | 1 per session |
| __application__ | Tạo 1 instance cho mỗi __ServletContext__ | 1 per context |

<br />

### 2.2. Singleton Scope (Mặc định)

```java
@Service
@Scope("singleton")  // Mặc định, có thể bỏ qua
public class UserService {
    // Chỉ có DUY NHẤT 1 instance trong toàn bộ ứng dụng
    // Mọi nơi inject UserService đều dùng chung 1 instance
}
```

```java
@RestController
public class UserController {
    @Autowired
    private UserService userService; // instance A
}

@RestController
public class AdminController {
    @Autowired
    private UserService userService; // cùng instance A
}
```

> __Lưu ý__: Vì singleton Bean được chia sẻ, nó phải __thread-safe__. Tránh lưu trạng thái (state) trong singleton Bean.

<br />

### 2.3. Prototype Scope

```java
@Component
@Scope("prototype")
public class ShoppingCart {
    private List<Item> items = new ArrayList<>();

    public void addItem(Item item) {
        items.add(item);
    }
}
```

Mỗi lần inject hoặc gọi `getBean()`, Spring sẽ tạo __instance mới__:
```java
@Service
public class OrderService {
    @Autowired
    private ApplicationContext context;

    public ShoppingCart createCart() {
        // Mỗi lần gọi sẽ tạo ShoppingCart mới
        return context.getBean(ShoppingCart.class);
    }
}
```

> __Lưu ý__: Spring __không quản lý toàn bộ lifecycle__ của prototype Bean. Sau khi tạo và inject, Bean không được Container theo dõi nữa. Method `@PreDestroy` sẽ __không được gọi__.

<br />

### 2.4. Request và Session Scope (Web Application)

```java
@Component
@Scope(value = WebApplicationContext.SCOPE_REQUEST, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class RequestScopedBean {
    // Tạo mới cho mỗi HTTP request
}

@Component
@Scope(value = WebApplicationContext.SCOPE_SESSION, proxyMode = ScopedProxyMode.TARGET_CLASS)
public class SessionScopedBean {
    // Tạo mới cho mỗi HTTP session
}
```

## 3. Bean Lifecycle

### 3.1. Vòng đời của Bean

Vòng đời của một Spring Bean trải qua các giai đoạn sau:

1. __Instantiation__ — Container tạo instance của Bean.
2. __Populate Properties__ — Inject các dependencies.
3. __BeanNameAware__ — Gọi `setBeanName()` nếu Bean implement `BeanNameAware`.
4. __BeanFactoryAware__ — Gọi `setBeanFactory()` nếu Bean implement `BeanFactoryAware`.
5. __Pre-initialization (BeanPostProcessor)__ — Gọi `postProcessBeforeInitialization()`.
6. __InitializingBean__ — Gọi `afterPropertiesSet()` nếu implement `InitializingBean`.
7. __Custom init method__ — Gọi method được chỉ định bởi `@PostConstruct` hoặc `initMethod`.
8. __Post-initialization (BeanPostProcessor)__ — Gọi `postProcessAfterInitialization()`.
9. __Bean is ready__ — Bean sẵn sàng sử dụng.
10. __Pre-destroy__ — Gọi `@PreDestroy` hoặc `destroy()`.
11. __Custom destroy method__ — Gọi method được chỉ định bởi `destroyMethod`.

<br />

### 3.2. `@PostConstruct` và `@PreDestroy`

Đây là cách phổ biến nhất để thực hiện logic __khởi tạo__ và __dọn dẹp__ tài nguyên.

```java
@Service
public class DatabaseService {

    private Connection connection;

    @PostConstruct
    public void init() {
        // Được gọi SAU KHI Bean được tạo và inject dependencies
        System.out.println("Initializing database connection...");
        this.connection = createConnection();
    }

    @PreDestroy
    public void cleanup() {
        // Được gọi TRƯỚC KHI Bean bị hủy
        System.out.println("Closing database connection...");
        if (connection != null) {
            connection.close();
        }
    }
}
```

<br />

### 3.3. Sử dụng `@Bean` với initMethod và destroyMethod

```java
@Configuration
public class AppConfig {

    @Bean(initMethod = "init", destroyMethod = "cleanup")
    public DatabaseService databaseService() {
        return new DatabaseService();
    }
}
```

<br />

### 3.4. Implement `InitializingBean` và `DisposableBean`

```java
@Service
public class CacheService implements InitializingBean, DisposableBean {

    @Override
    public void afterPropertiesSet() throws Exception {
        // Tương tự @PostConstruct
        System.out.println("Cache initialized");
    }

    @Override
    public void destroy() throws Exception {
        // Tương tự @PreDestroy
        System.out.println("Cache cleared");
    }
}
```

> __Thứ tự ưu tiên__ khi khởi tạo: `@PostConstruct` → `InitializingBean.afterPropertiesSet()` → `initMethod`

## 4. Component Scanning

### 4.1. Component Scanning là gì?

__Component Scanning__ là cơ chế tự động phát hiện và đăng ký các Bean. Spring sẽ quét các package để tìm các class được đánh dấu bằng __Stereotype Annotations__ (`@Component`, `@Service`, `@Repository`, `@Controller`).

<br />

### 4.2. Cách hoạt động

`@SpringBootApplication` đã bao gồm `@ComponentScan`, tự động quét __package chứa main class__ và tất cả __sub-packages__.

```
com.example.myapp/              ← @SpringBootApplication ở đây
   ├──controller/               ← Được quét ✅
   ├──service/                  ← Được quét ✅
   ├──repository/               ← Được quét ✅
   └──config/                   ← Được quét ✅

com.example.other/              ← KHÔNG được quét ❌
```

Nếu muốn quét thêm package khác:
```java
@SpringBootApplication
@ComponentScan(basePackages = {"com.example.myapp", "com.example.other"})
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

## 5. Câu hỏi phỏng vấn thường gặp

__Q: Sự khác biệt giữa `@Component` và `@Service`?__

Về mặt kỹ thuật, `@Service` kế thừa từ `@Component` nên __không có sự khác biệt về chức năng__. `@Service` mang ý nghĩa ngữ cảnh (semantic) để đánh dấu class thuộc tầng business logic. Tuy nhiên, Spring có thể sử dụng các annotation này để áp dụng AOP hoặc xử lý đặc biệt trong tương lai.

__Q: Singleton Bean có phải là Singleton Pattern không?__

__Không hoàn toàn__. Singleton Pattern đảm bảo chỉ có 1 instance trong toàn bộ JVM. Singleton Bean đảm bảo chỉ có 1 instance __trong mỗi IoC Container__. Nếu có nhiều Container, sẽ có nhiều instance.

__Q: Khi nào dùng Prototype scope?__

Dùng khi Bean có __trạng thái (stateful)__ và mỗi consumer cần một instance riêng, ví dụ: ShoppingCart, FormData, Request-specific processors.
