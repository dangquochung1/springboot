# Testing trong Spring Boot

<br />

## 1. Tổng quan về Testing

### 1.1. Tại sao cần Testing?

Testing giúp:
- __Phát hiện bug sớm__ trước khi deploy lên production.
- __Đảm bảo code hoạt động đúng__ khi refactor hoặc thêm tính năng mới.
- __Tài liệu sống__ — test cases mô tả cách code hoạt động.
- __Tăng sự tự tin__ khi deploy.

<br />

### 1.2. Các loại Testing

| Loại | Phạm vi | Tốc độ | Mục đích |
| ---- | ------- | ------ | -------- |
| __Unit Test__ | 1 class/method | Rất nhanh | Kiểm tra logic riêng lẻ |
| __Integration Test__ | Nhiều components | Chậm hơn | Kiểm tra sự phối hợp giữa các components |
| __End-to-End Test__ | Toàn bộ ứng dụng | Chậm nhất | Kiểm tra luồng hoàn chỉnh |

<br />

### 1.3. Dependencies

`spring-boot-starter-test` đã bao gồm:
- __JUnit 5__ (Jupiter) — Test framework
- __Mockito__ — Mocking framework
- __AssertJ__ — Fluent assertions
- __Spring Test__ — Spring testing utilities
- __MockMvc__ — Test Spring MVC controllers

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

## 2. Unit Test

### 2.1. Unit Test với Mockito

Unit test kiểm tra __1 class duy nhất__, các dependency được __mock__ (giả lập).

__UserServiceTest.java__
```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private UserServiceImpl userService;

    private User testUser;

    @BeforeEach
    void setUp() {
        testUser = new User();
        testUser.setId(1L);
        testUser.setName("Nguyen Van A");
        testUser.setEmail("a@example.com");
    }

    @Test
    @DisplayName("Should return user when found by ID")
    void findById_WhenUserExists_ReturnsUser() {
        // Given (Arrange)
        when(userRepository.findById(1L)).thenReturn(Optional.of(testUser));

        // When (Act)
        User result = userService.findById(1L);

        // Then (Assert)
        assertNotNull(result);
        assertEquals("Nguyen Van A", result.getName());
        assertEquals("a@example.com", result.getEmail());
        verify(userRepository, times(1)).findById(1L);
    }

    @Test
    @DisplayName("Should throw exception when user not found")
    void findById_WhenUserNotExists_ThrowsException() {
        // Given
        when(userRepository.findById(99L)).thenReturn(Optional.empty());

        // When & Then
        assertThrows(ResourceNotFoundException.class, () -> {
            userService.findById(99L);
        });
        verify(userRepository, times(1)).findById(99L);
    }

    @Test
    @DisplayName("Should save user successfully")
    void save_ValidUser_ReturnsSavedUser() {
        // Given
        when(userRepository.save(any(User.class))).thenReturn(testUser);

        // When
        User result = userService.save(testUser);

        // Then
        assertNotNull(result);
        assertEquals(1L, result.getId());
        verify(userRepository).save(testUser);
    }

    @Test
    @DisplayName("Should return all users")
    void findAll_ReturnsUserList() {
        // Given
        User user2 = new User();
        user2.setId(2L);
        user2.setName("Tran Van B");
        when(userRepository.findAll()).thenReturn(List.of(testUser, user2));

        // When
        List<User> result = userService.findAll();

        // Then
        assertEquals(2, result.size());
        verify(userRepository).findAll();
    }
}
```

### 2.2. Mockito Annotations

| Annotation | Mô tả |
| ---------- | ----- |
| `@Mock` | Tạo mock object (giả lập) |
| `@InjectMocks` | Tạo instance thật và inject các mock vào |
| `@Spy` | Tạo spy (partial mock — gọi method thật trừ khi bị stub) |
| `@Captor` | Capture argument được truyền vào mock |

### 2.3. Mockito Methods

| Method | Mô tả |
| ------ | ----- |
| `when().thenReturn()` | Khi gọi method X → trả về Y |
| `when().thenThrow()` | Khi gọi method X → ném exception |
| `verify()` | Kiểm tra method có được gọi không |
| `verify(mock, times(n))` | Kiểm tra method được gọi đúng n lần |
| `verify(mock, never())` | Kiểm tra method KHÔNG được gọi |
| `any()`, `anyLong()` | Argument matcher — khớp bất kỳ giá trị nào |

## 3. Integration Test

### 3.1. `@SpringBootTest`

`@SpringBootTest` load __toàn bộ ApplicationContext__, phù hợp cho integration test.

```java
@SpringBootTest
@AutoConfigureMockMvc
@Transactional  // Rollback sau mỗi test
class UserIntegrationTest {

    @Autowired
    private MockMvc mockMvc;

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private ObjectMapper objectMapper;

    @BeforeEach
    void setUp() {
        User user = new User();
        user.setName("Test User");
        user.setEmail("test@example.com");
        user.setPassword("password");
        userRepository.save(user);
    }

    @Test
    void getUser_ShouldReturnUser() throws Exception {
        mockMvc.perform(get("/api/users/1")
                        .contentType(MediaType.APPLICATION_JSON))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.name").value("Test User"))
                .andExpect(jsonPath("$.email").value("test@example.com"));
    }

    @Test
    void createUser_ShouldReturn201() throws Exception {
        UserRequest request = new UserRequest("New User", "new@example.com", "password");

        mockMvc.perform(post("/api/users")
                        .contentType(MediaType.APPLICATION_JSON)
                        .content(objectMapper.writeValueAsString(request)))
                .andExpect(status().isCreated())
                .andExpect(jsonPath("$.name").value("New User"));
    }
}
```

### 3.2. `@WebMvcTest` — Test Controller Layer

`@WebMvcTest` chỉ load __Controller layer__, mock các dependencies.

```java
@WebMvcTest(UserController.class)
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean  // Mock bean trong Spring Context
    private UserService userService;

    @Autowired
    private ObjectMapper objectMapper;

    @Test
    void getUser_ShouldReturnUser() throws Exception {
        // Given
        UserResponse response = new UserResponse(1L, "Test User", "test@example.com");
        when(userService.findById(1L)).thenReturn(response);

        // When & Then
        mockMvc.perform(get("/api/users/1")
                        .contentType(MediaType.APPLICATION_JSON))
                .andExpect(status().isOk())
                .andExpect(jsonPath("$.id").value(1))
                .andExpect(jsonPath("$.name").value("Test User"))
                .andExpect(jsonPath("$.email").value("test@example.com"));
    }

    @Test
    void getUser_NotFound_ShouldReturn404() throws Exception {
        when(userService.findById(99L))
                .thenThrow(new ResourceNotFoundException("User not found"));

        mockMvc.perform(get("/api/users/99"))
                .andExpect(status().isNotFound());
    }

    @Test
    void createUser_InvalidInput_ShouldReturn400() throws Exception {
        UserRequest invalidRequest = new UserRequest("", "", ""); // Invalid

        mockMvc.perform(post("/api/users")
                        .contentType(MediaType.APPLICATION_JSON)
                        .content(objectMapper.writeValueAsString(invalidRequest)))
                .andExpect(status().isBadRequest());
    }
}
```

### 3.3. `@DataJpaTest` — Test Repository Layer

`@DataJpaTest` chỉ load __JPA components__, sử dụng __embedded database__ (H2).

```java
@DataJpaTest
class UserRepositoryTest {

    @Autowired
    private UserRepository userRepository;

    @Autowired
    private TestEntityManager entityManager;

    @Test
    void findByEmail_ShouldReturnUser() {
        // Given
        User user = new User();
        user.setName("Test User");
        user.setEmail("test@example.com");
        user.setPassword("password");
        entityManager.persistAndFlush(user);

        // When
        Optional<User> found = userRepository.findByEmail("test@example.com");

        // Then
        assertTrue(found.isPresent());
        assertEquals("Test User", found.get().getName());
    }

    @Test
    void findByEmail_NotFound_ShouldReturnEmpty() {
        Optional<User> found = userRepository.findByEmail("nonexistent@example.com");
        assertTrue(found.isEmpty());
    }
}
```

### 3.4. So sánh các Test Annotations

| Annotation | Load | Tốc độ | Dùng cho |
| ---------- | ---- | ------ | -------- |
| `@ExtendWith(MockitoExtension.class)` | Không load Spring | Rất nhanh | Unit test (Service) |
| `@WebMvcTest` | Chỉ Web layer | Nhanh | Controller test |
| `@DataJpaTest` | Chỉ JPA layer | Nhanh | Repository test |

## 5. Best Practices

- __Quy tắc đặt tên test__: `methodName_condition_expectedResult` (ví dụ: `findById_WhenUserExists_ReturnsUser`).
- __AAA Pattern__: Arrange (chuẩn bị) → Act (thực thi) → Assert (kiểm tra).
- __Mỗi test chỉ kiểm tra 1 điều__.
- __Test cả happy path và error cases__.
- __Dùng `@DisplayName`__ để mô tả test dễ đọc.
- __Unit test nhanh, không phụ thuộc database/network__.

## 6. Câu hỏi phỏng vấn thường gặp

__Q: `@Mock` vs `@MockBean`?__

`@Mock` (Mockito) tạo mock object thuần túy, __không liên quan đến Spring__. `@MockBean` (Spring Boot Test) tạo mock và __đưa vào Spring ApplicationContext__, thay thế Bean thật.

__Q: Tại sao dùng `@Transactional` trong test?__

Để __tự động rollback__ sau mỗi test, đảm bảo các test __độc lập__ với nhau, không ảnh hưởng dữ liệu lẫn nhau.

__Q: `@WebMvcTest` vs `@SpringBootTest`?__

`@WebMvcTest` chỉ load Controller layer → nhanh, phù hợp test API endpoints. `@SpringBootTest` load toàn bộ context → chậm hơn, phù hợp integration test end-to-end.
