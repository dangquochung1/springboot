# Spring IoC Container và Dependency Injection

<br />

## 1. Inversion of Control (IoC)

### 1.1. IoC là gì?

__Inversion of Control (IoC)__ — Đảo ngược quyền điều khiển — là một nguyên lý thiết kế trong đó quyền kiểm soát việc tạo và quản lý đối tượng được __chuyển giao__ từ code của lập trình viên sang __framework__ (container).

Trong lập trình truyền thống, chính code của chúng ta tạo ra các đối tượng phụ thuộc:

```java
// Không dùng IoC — tự tạo đối tượng
public class UserService {
    private UserRepository userRepository = new UserRepositoryImpl(); // tight coupling

    public User findUser(Long id) {
        return userRepository.findById(id);
    }
}
```

Vấn đề của cách tiếp cận trên:
- __Tight coupling__: `UserService` phụ thuộc trực tiếp vào `UserRepositoryImpl`.
- Không thể thay thế implementation mà không sửa code.
- Khó viết __unit test__ (không thể mock `UserRepository`).

<br />

Với __IoC__, framework sẽ chịu trách nhiệm tạo và cung cấp các đối tượng:

```java
// Dùng IoC — framework quản lý đối tượng
@Service
public class UserService {
    private final UserRepository userRepository; // loose coupling

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository; // framework inject vào
    }

    public User findUser(Long id) {
        return userRepository.findById(id);
    }
}
```

<br />

### 1.2. IoC Container

__IoC Container__ (còn gọi là __Spring Container__) là thành phần cốt lõi của Spring Framework, chịu trách nhiệm:

- __Tạo__ các đối tượng (gọi là __Bean__).
- __Quản lý lifecycle__ của các Bean.
- __Inject__ (tiêm) các dependency vào Bean.
- __Cấu hình__ và lắp ráp các Bean với nhau.

Spring cung cấp 2 loại IoC Container:

| Container | Interface | Mô tả |
| --------- | --------- | ----- |
| __BeanFactory__ | `org.springframework.beans.factory.BeanFactory` | Container cơ bản, hỗ trợ __lazy loading__ |
| __ApplicationContext__ | `org.springframework.context.ApplicationContext` | Mở rộng từ BeanFactory, thêm nhiều tính năng enterprise |

<br />

### 1.3. BeanFactory vs ApplicationContext

| Tiêu chí | BeanFactory | ApplicationContext |
| -------- | ----------- | ------------------ |
| __Bean loading__ | Lazy loading (tạo bean khi được yêu cầu) | Eager loading (tạo bean khi khởi động) |
| __Event handling__ | Không hỗ trợ | Hỗ trợ event publishing |
| __Internationalization__ | Không hỗ trợ | Hỗ trợ i18n |
| __AOP__ | Không tích hợp | Tích hợp sẵn |
| __Annotation support__ | Hạn chế | Đầy đủ |
| __Sử dụng__ | Ứng dụng nhỏ, tài nguyên hạn chế | __Hầu hết các ứng dụng__ (recommended) |

Trong Spring Boot, __ApplicationContext__ được sử dụng mặc định.

<br />

## 2. Dependency Injection (DI)

### 2.1. DI là gì?

__Dependency Injection (DI)__ — Tiêm phụ thuộc — là một __design pattern__ và cũng là cách hiện thực hóa nguyên lý __IoC__. Thay vì đối tượng tự tạo ra các dependency của mình, các dependency được __"tiêm" (inject)__ vào từ bên ngoài bởi IoC Container.

```
IoC = Nguyên lý (Principle)
DI  = Kỹ thuật thực hiện IoC (Implementation)
```

<br />

### 2.2. Các loại Dependency Injection

Spring hỗ trợ 3 cách inject dependency:

#### 2.2.1. Constructor Injection __(Recommended)__

Dependency được inject qua __constructor__ của class.

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    // Constructor Injection
    // @Autowired là tùy chọn khi class chỉ có 1 constructor
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public User findUser(Long id) {
        return userRepository.findById(id).orElse(null);
    }
}
```

__Ưu điểm:__
- Dependency được khai báo __final__ → đảm bảo __immutability__.
- Đảm bảo dependency __không null__ khi khởi tạo.
- Dễ viết __unit test__ (truyền mock qua constructor).
- Phát hiện __circular dependency__ sớm (khi khởi động).

<br />

#### 2.2.2. Setter Injection

Dependency được inject qua __setter method__.

```java
@Service
public class UserService {

    private UserRepository userRepository;

    // Setter Injection
    @Autowired
    public void setUserRepository(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public User findUser(Long id) {
        return userRepository.findById(id).orElse(null);
    }
}
```

__Ưu điểm:__
- Phù hợp cho __optional dependencies__.
- Cho phép thay đổi dependency sau khi khởi tạo.

__Nhược điểm:__
- Dependency có thể bị __null__ nếu setter chưa được gọi.
- Không đảm bảo __immutability__.

<br />

#### 2.2.3. Field Injection __(Không khuyến khích)__

Dependency được inject trực tiếp vào __field__ bằng `@Autowired`.

```java
@Service
public class UserService {

    @Autowired  // Field Injection
    private UserRepository userRepository;

    public User findUser(Long id) {
        return userRepository.findById(id).orElse(null);
    }
}
```

__Nhược điểm:__
- Khó viết __unit test__ (phải dùng Reflection).
- Không thể khai báo __final__.
- Ẩn các dependency, khó nhận biết class phụ thuộc vào gì.

<br />

### 2.3. So sánh 3 loại DI

| Tiêu chí | Constructor | Setter | Field |
| -------- | ----------- | ------ | ----- |
| __Immutability__ | ✅ Hỗ trợ (final) | ❌ | ❌ |
| __Required dependency__ | ✅ Đảm bảo | ❌ Có thể null | ❌ Có thể null |
| __Testability__ | ✅ Dễ test | ✅ Dễ test | ❌ Khó test |
| __Circular dependency__ | Phát hiện sớm | Cho phép | Cho phép |
| __Recommended__ | ✅ __Yes__ | Cho optional deps | ❌ Avoid |

> __Best Practice__: Luôn ưu tiên sử dụng __Constructor Injection__ cho mandatory dependencies.

<br />

## 3. Các Annotation quan trọng cho DI

### 3.1. `@Autowired`

`@Autowired` yêu cầu Spring tự động inject dependency. Có thể dùng trên constructor, setter, hoặc field.

```java
@Service
public class OrderService {

    private final UserService userService;
    private final ProductService productService;

    @Autowired  // Tùy chọn khi chỉ có 1 constructor
    public OrderService(UserService userService, ProductService productService) {
        this.userService = userService;
        this.productService = productService;
    }
}
```

> __Lưu ý__: Từ Spring 4.3+, nếu class chỉ có __duy nhất một constructor__, `@Autowired` có thể được __bỏ qua__.

<br />

### 3.2. `@Qualifier`

Khi có __nhiều Bean cùng kiểu__, `@Qualifier` chỉ định chính xác Bean nào sẽ được inject.

```java
public interface NotificationService {
    void send(String message);
}

@Service("emailService")
public class EmailNotificationService implements NotificationService {
    @Override
    public void send(String message) {
        System.out.println("Email: " + message);
    }
}

@Service("smsService")
public class SmsNotificationService implements NotificationService {
    @Override
    public void send(String message) {
        System.out.println("SMS: " + message);
    }
}

@Service
public class OrderService {
    private final NotificationService notificationService;

    // Dùng @Qualifier để chọn implementation cụ thể
    public OrderService(@Qualifier("emailService") NotificationService notificationService) {
        this.notificationService = notificationService;
    }
}
```

<br />

### 3.3. `@Primary`

`@Primary` đánh dấu Bean __mặc định__ khi có nhiều Bean cùng kiểu.

```java
@Service
@Primary  // Bean mặc định
public class EmailNotificationService implements NotificationService {
    @Override
    public void send(String message) {
        System.out.println("Email: " + message);
    }
}

@Service
public class SmsNotificationService implements NotificationService {
    @Override
    public void send(String message) {
        System.out.println("SMS: " + message);
    }
}

@Service
public class OrderService {
    private final NotificationService notificationService;

    // Sẽ inject EmailNotificationService (vì có @Primary)
    public OrderService(NotificationService notificationService) {
        this.notificationService = notificationService;
    }
}
```

> __Lưu ý__: `@Qualifier` có __độ ưu tiên cao hơn__ `@Primary`.

<br />

### 3.4. `@Value`

`@Value` inject giá trị từ __properties file__ hoặc __environment variables__.

```java
@Service
public class AppConfigService {

    @Value("${app.name}")
    private String appName;

    @Value("${server.port:8080}")  // Giá trị mặc định là 8080
    private int serverPort;

    @Value("${app.feature.enabled:false}")
    private boolean featureEnabled;
}
```

## 4. Ví dụ tổng hợp

Chúng ta sẽ tạo một ví dụ minh họa đầy đủ về DI trong Spring Boot:

__UserRepository.java__
```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
    Optional<User> findByEmail(String email);
}
```

__UserService.java__
```java
public interface UserService {
    UserResponse getUserById(Long id);
    List<UserResponse> getAllUsers();
}
```

__UserServiceImpl.java__
```java
@Service
@RequiredArgsConstructor  // Lombok tự tạo constructor injection
public class UserServiceImpl implements UserService {

    private final UserRepository userRepository;

    @Override
    public UserResponse getUserById(Long id) {
        User user = userRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException("User not found"));
        return mapToResponse(user);
    }

    @Override
    public List<UserResponse> getAllUsers() {
        return userRepository.findAll()
                .stream()
                .map(this::mapToResponse)
                .collect(Collectors.toList());
    }

    private UserResponse mapToResponse(User user) {
        return new UserResponse(user.getId(), user.getName(), user.getEmail());
    }
}
```

__UserController.java__
```java
@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;  // Constructor injection qua Lombok

    @GetMapping("/{id}")
    public ResponseEntity<UserResponse> getUser(@PathVariable Long id) {
        return ResponseEntity.ok(userService.getUserById(id));
    }

    @GetMapping
    public ResponseEntity<List<UserResponse>> getAllUsers() {
        return ResponseEntity.ok(userService.getAllUsers());
    }
}
```

```
src/main/java/com/example/demo/
   ├──DemoApplication.java
   ├──controller/
   │  └──UserController.java
   ├──service/
   │  ├──UserService.java
   │  └──impl/
   │     └──UserServiceImpl.java
   ├──repository/
   │  └──UserRepository.java
   └──model/
      └──User.java
```
