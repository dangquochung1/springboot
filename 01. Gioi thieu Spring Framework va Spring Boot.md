# Giới thiệu Spring Framework và Spring Boot

<br />

## 1. Spring Framework

### 1.1. Spring Framework là gì?

__Spring Framework__ là một framework mã nguồn mở dành cho ngôn ngữ lập trình Java, được phát triển bởi __Rod Johnson__ và ra mắt lần đầu vào năm 2003. Spring cung cấp một mô hình lập trình và cấu hình toàn diện cho các ứng dụng doanh nghiệp (enterprise) dựa trên nền tảng Java.

Spring Framework giải quyết nhiều vấn đề phức tạp trong phát triển ứng dụng Java EE, giúp lập trình viên tập trung vào __business logic__ thay vì phải xử lý các __boilerplate code__ lặp đi lặp lại.

<br />

### 1.2. Tại sao cần Spring?

Trước khi có Spring, việc phát triển ứng dụng Java EE gặp nhiều khó khăn:

- Các đối tượng bị __tight coupling__ (liên kết chặt chẽ), khó thay đổi và kiểm thử.
- Phải viết rất nhiều __boilerplate code__ cho cấu hình XML, quản lý transaction, kết nối database,...
- Khó __unit test__ vì các class phụ thuộc trực tiếp vào nhau.
- Phải tự quản lý __lifecycle__ của các đối tượng.

Spring Framework giải quyết tất cả vấn đề trên thông qua hai nguyên lý cốt lõi: __Inversion of Control (IoC)__ và __Dependency Injection (DI)__.

<br />

### 1.3. Các module chính của Spring Framework

| Module | Mô tả |
| ------ | ----- |
| __Spring Core__ | Cung cấp IoC Container, DI — nền tảng của toàn bộ framework |
| __Spring AOP__ | Hỗ trợ lập trình hướng khía cạnh (Aspect-Oriented Programming) |
| __Spring MVC__ | Framework cho phát triển web application theo mô hình MVC |
| __Spring Data__ | Đơn giản hóa việc truy cập dữ liệu (JPA, MongoDB, Redis,...) |
| __Spring Security__ | Cung cấp xác thực (authentication) và phân quyền (authorization) |
| __Spring Transaction__ | Quản lý transaction một cách nhất quán |
| __Spring Test__ | Hỗ trợ unit test và integration test |

<br />

## 2. Spring Boot

### 2.1. Spring Boot là gì?

__Spring Boot__ là một dự án con (sub-project) của Spring Framework, được phát triển bởi __Pivotal Team__. Spring Boot giúp tạo ra các ứng dụng Spring __dễ dàng__ và __nhanh chóng__ hơn bằng cách loại bỏ phần lớn cấu hình thủ công.

Spring Boot __không phải__ là một framework riêng biệt, mà là một cách tiếp cận để xây dựng ứng dụng Spring nhanh hơn, với ít cấu hình hơn.

<br />

### 2.2. So sánh Spring Framework vs Spring Boot

| Tiêu chí | Spring Framework | Spring Boot |
| -------- | ---------------- | ----------- |
| __Cấu hình__ | Phải cấu hình thủ công (XML/Java Config) | Auto-configuration |
| __Dependencies__ | Phải khai báo từng dependency | Starter dependencies gom nhóm sẵn |
| __Server__ | Cần deploy lên external server | Embedded server (Tomcat, Jetty) |
| __Khởi tạo project__ | Phức tạp, nhiều bước | Nhanh chóng qua Spring Initializr |
| __Boilerplate code__ | Nhiều | Rất ít |
| __Production-ready__ | Tự cấu hình | Actuator, Health check tích hợp sẵn |

## 3. Các tính năng quan trọng của Spring Boot

### 3.1. Auto-configuration

__Auto-configuration__ là tính năng cốt lõi nhất của Spring Boot. Nó tự động cấu hình ứng dụng Spring dựa trên các __JAR dependencies__ có trong __classpath__.

Ví dụ:
- Khi thêm `spring-boot-starter-web` → tự động cấu hình __DispatcherServlet__, __Embedded Tomcat__.
- Khi thêm `spring-boot-starter-data-jpa` → tự động cấu hình __DataSource__, __EntityManager__.
- Khi thêm `spring-boot-starter-security` → tự động cấu hình __Basic Authentication__.

```java
@SpringBootApplication
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

Nếu muốn loại trừ một auto-configuration cụ thể:
```java
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class})
public class MyApplication {
    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

<br />

### 3.2. Spring Boot Starters

__Starters__ là các bộ mô tả phụ thuộc giúp gom nhóm các thư viện liên quan.

| Starter | Mô tả |
| ------- | ----- |
| `spring-boot-starter-web` | Web application với Spring MVC, REST, Embedded Tomcat |
| `spring-boot-starter-data-jpa` | Spring Data JPA với Hibernate |
| `spring-boot-starter-security` | Spring Security |
| `spring-boot-starter-test` | Testing (JUnit, Mockito, Spring Test) |
| `spring-boot-starter-validation` | Bean Validation (Hibernate Validator) |
| `spring-boot-starter-actuator` | Production-ready features |

Ví dụ trong __pom.xml__:
```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
</dependencies>
```

<br />

### 3.3. Annotation `@SpringBootApplication`

`@SpringBootApplication` là annotation __tổ hợp__ bao gồm 3 annotation:

| Annotation | Tác dụng |
| ---------- | -------- |
| `@SpringBootConfiguration` | Đánh dấu class là nguồn cấu hình (tương đương `@Configuration`) |
| `@EnableAutoConfiguration` | Kích hoạt cơ chế auto-configuration |
| `@ComponentScan` | Tự động quét và đăng ký các Bean trong package hiện tại và sub-packages |

<br />

### 3.4. Embedded Server

Spring Boot tích hợp sẵn __Embedded Server__: __Tomcat__ (mặc định), __Jetty__, __Undertow__.

Để chuyển từ Tomcat sang Jetty:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-tomcat</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-jetty</artifactId>
</dependency>
```

## 4. Cấu trúc Project Spring Boot

### 4.1. Cấu trúc thư mục chuẩn

```
my-spring-boot-app/
   ├──src/
   │  ├──main/
   │  │  ├──java/
   │  │  │  └──com/example/myapp/
   │  │  │     ├──MyApplication.java
   │  │  │     ├──controller/
   │  │  │     ├──service/
   │  │  │     ├──repository/
   │  │  │     ├──model/
   │  │  │     ├──dto/
   │  │  │     ├──config/
   │  │  │     └──exception/
   │  │  └──resources/
   │  │     ├──application.properties
   │  │     ├──static/
   │  │     └──templates/
   │  └──test/
   │     └──java/
   └──pom.xml
```

<br />

### 4.2. File `application.properties`

```properties
# Server
server.port=8080

# Database
spring.datasource.url=jdbc:mysql://localhost:3306/mydb
spring.datasource.username=root
spring.datasource.password=password

# JPA / Hibernate
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

<br />

### 4.3. Spring Profiles

__Spring Profiles__ cho phép cấu hình khác nhau cho từng môi trường (dev, test, prod).

```properties
# application.properties
spring.profiles.active=dev
```

Hoặc khi chạy:
```bash
java -jar myapp.jar --spring.profiles.active=prod
```

## 5. Spring Boot DevTools và Actuator

### 5.1. DevTools

__Spring Boot DevTools__ hỗ trợ: __Automatic Restart__, __LiveReload__, __Disable Caching__.

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <scope>runtime</scope>
    <optional>true</optional>
</dependency>
```

### 5.2. Actuator

__Spring Boot Actuator__ cung cấp các endpoint giám sát ứng dụng.

| Endpoint | Mô tả |
| -------- | ----- |
| `/actuator/health` | Kiểm tra sức khỏe ứng dụng |
| `/actuator/info` | Thông tin ứng dụng |
| `/actuator/metrics` | Metrics (CPU, memory, requests,...) |
| `/actuator/beans` | Danh sách tất cả beans |

```properties
management.endpoints.web.exposure.include=*
management.endpoint.health.show-details=always
```
