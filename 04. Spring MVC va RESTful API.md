# Spring MVC và RESTful API

<br />

## 1. Spring MVC

### 1.1. Spring MVC là gì?

__Spring MVC__ (Model-View-Controller) là một module của Spring Framework dùng để xây dựng __web application__ theo mô hình __MVC__. Spring MVC cung cấp một kiến trúc linh hoạt và mạnh mẽ để xử lý HTTP request và trả về response.

Các thành phần chính:
- __Model__: Dữ liệu của ứng dụng (Entity, DTO).
- __View__: Giao diện hiển thị (Thymeleaf, JSP, JSON/XML).
- __Controller__: Xử lý request, điều phối giữa Model và View.

<br />

### 1.2. Luồng xử lý Request trong Spring MVC

```
Client Request → DispatcherServlet → HandlerMapping → Controller
                                                          ↓
Client Response ← ViewResolver ← View ← Model ← Controller
```

1. __DispatcherServlet__ nhận HTTP request từ client.
2. __HandlerMapping__ xác định Controller nào sẽ xử lý request.
3. __Controller__ thực hiện business logic và trả về Model + View name.
4. __ViewResolver__ tìm View tương ứng với view name.
5. __View__ render dữ liệu từ Model và trả về response cho client.

<br />

### 1.3. DispatcherServlet

__DispatcherServlet__ là __Front Controller__ của Spring MVC, đóng vai trò trung tâm nhận mọi HTTP request và chuyển tiếp đến Controller phù hợp.

Trong Spring Boot, __DispatcherServlet__ được tự động cấu hình khi thêm `spring-boot-starter-web`.

## 2. Controller

### 2.1. `@Controller` vs `@RestController`

| Annotation | Trả về | Dùng khi |
| ---------- | ------ | -------- |
| `@Controller` | __View name__ (HTML page) | Server-side rendering (Thymeleaf, JSP) |
| `@RestController` | __Object__ (tự động serialize thành JSON/XML) | RESTful API |

`@RestController` = `@Controller` + `@ResponseBody`

```java
// Trả về HTML page
@Controller
public class PageController {

    @GetMapping("/home")
    public String homePage(Model model) {
        model.addAttribute("message", "Hello World");
        return "home";  // Trả về view name → ViewResolver tìm home.html
    }
}
```

```java
// Trả về JSON
@RestController
@RequestMapping("/api")
public class ApiController {

    @GetMapping("/hello")
    public Map<String, String> hello() {
        return Map.of("message", "Hello World");
        // Tự động serialize thành JSON: {"message": "Hello World"}
    }
}
```

<br />

### 2.2. Request Mapping Annotations

| Annotation | HTTP Method | Ý nghĩa |
| ---------- | ----------- | -------- |
| `@RequestMapping` | Tất cả (hoặc chỉ định) | Annotation tổng quát |
| `@GetMapping` | GET | Lấy dữ liệu |
| `@PostMapping` | POST | Tạo mới dữ liệu |
| `@PutMapping` | PUT | Cập nhật toàn bộ |
| `@PatchMapping` | PATCH | Cập nhật một phần |
| `@DeleteMapping` | DELETE | Xóa dữ liệu |

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping                    // GET /api/users
    public List<User> getAll() { ... }

    @GetMapping("/{id}")           // GET /api/users/1
    public User getById(@PathVariable Long id) { ... }

    @PostMapping                   // POST /api/users
    public User create(@RequestBody User user) { ... }

    @PutMapping("/{id}")           // PUT /api/users/1
    public User update(@PathVariable Long id, @RequestBody User user) { ... }

    @DeleteMapping("/{id}")        // DELETE /api/users/1
    public void delete(@PathVariable Long id) { ... }
}
```

## 3. Nhận dữ liệu từ Request

### 3.1. `@PathVariable`

Lấy giá trị từ __URL path__.

```java
// GET /api/users/5
@GetMapping("/api/users/{id}")
public User getUser(@PathVariable Long id) {
    // id = 5
    return userService.findById(id);
}

// GET /api/users/5/orders/10
@GetMapping("/api/users/{userId}/orders/{orderId}")
public Order getOrder(@PathVariable Long userId, @PathVariable Long orderId) {
    return orderService.findByUserAndOrder(userId, orderId);
}
```

<br />

### 3.2. `@RequestParam`

Lấy giá trị từ __query parameters__.

```java
// GET /api/users?page=0&size=10&sort=name
@GetMapping("/api/users")
public Page<User> getUsers(
        @RequestParam(defaultValue = "0") int page,
        @RequestParam(defaultValue = "10") int size,
        @RequestParam(required = false) String sort) {
    return userService.findAll(page, size, sort);
}
```

<br />

### 3.3. `@RequestBody`

Nhận dữ liệu từ __body__ của request (thường là JSON), tự động __deserialize__ thành Java object.

```java
// POST /api/users
// Body: {"name": "Nguyen Van A", "email": "a@example.com"}
@PostMapping("/api/users")
public ResponseEntity<User> createUser(@RequestBody @Valid UserRequest request) {
    User user = userService.create(request);
    return ResponseEntity.status(HttpStatus.CREATED).body(user);
}
```

__UserRequest.java__ (DTO)
```java
@Getter
@Setter
public class UserRequest {

    @NotBlank(message = "Name is required")
    private String name;

    @Email(message = "Invalid email format")
    @NotBlank(message = "Email is required")
    private String email;

    @Size(min = 6, message = "Password must be at least 6 characters")
    private String password;
}
```

<br />

### 3.4. `@RequestHeader`

Lấy giá trị từ __HTTP header__.

```java
@GetMapping("/api/profile")
public User getProfile(@RequestHeader("Authorization") String token) {
    return userService.findByToken(token);
}
```

## 4. ResponseEntity

### 4.1. ResponseEntity là gì?

__ResponseEntity__ cho phép kiểm soát hoàn toàn HTTP response bao gồm: __status code__, __headers__, và __body__.

```java
@RestController
@RequestMapping("/api/users")
@RequiredArgsConstructor
public class UserController {

    private final UserService userService;

    @GetMapping("/{id}")
    public ResponseEntity<UserResponse> getUser(@PathVariable Long id) {
        UserResponse user = userService.findById(id);
        return ResponseEntity.ok(user);  // 200 OK
    }

    @PostMapping
    public ResponseEntity<UserResponse> createUser(@RequestBody @Valid UserRequest request) {
        UserResponse user = userService.create(request);
        URI location = URI.create("/api/users/" + user.getId());
        return ResponseEntity
                .created(location)  // 201 Created + Location header
                .body(user);
    }

    @PutMapping("/{id}")
    public ResponseEntity<UserResponse> updateUser(
            @PathVariable Long id,
            @RequestBody @Valid UserRequest request) {
        UserResponse user = userService.update(id, request);
        return ResponseEntity.ok(user);  // 200 OK
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> deleteUser(@PathVariable Long id) {
        userService.delete(id);
        return ResponseEntity.noContent().build();  // 204 No Content
    }
}
```

<br />

### 4.2. Các HTTP Status Code phổ biến

| Status Code | Constant | Ý nghĩa |
| ----------- | -------- | -------- |
| 200 | `HttpStatus.OK` | Thành công |
| 201 | `HttpStatus.CREATED` | Tạo mới thành công |
| 204 | `HttpStatus.NO_CONTENT` | Thành công, không có body |
| 400 | `HttpStatus.BAD_REQUEST` | Request không hợp lệ |
| 401 | `HttpStatus.UNAUTHORIZED` | Chưa xác thực |
| 403 | `HttpStatus.FORBIDDEN` | Không có quyền |
| 404 | `HttpStatus.NOT_FOUND` | Không tìm thấy |
| 500 | `HttpStatus.INTERNAL_SERVER_ERROR` | Lỗi server |

## 5. Exception Handling

### 5.1. `@ControllerAdvice` và `@ExceptionHandler`

__`@ControllerAdvice`__ cho phép xử lý exception __tập trung__ cho toàn bộ ứng dụng thay vì xử lý trong từng Controller.

__ErrorResponse.java__ (DTO)
```java
@Getter
@Setter
@AllArgsConstructor
public class ErrorResponse {
    private int status;
    private String message;
    private LocalDateTime timestamp;
}
```

__ResourceNotFoundException.java__
```java
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String message) {
        super(message);
    }
}
```

__GlobalExceptionHandler.java__
```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleNotFound(ResourceNotFoundException ex) {
        ErrorResponse error = new ErrorResponse(
                HttpStatus.NOT_FOUND.value(),
                ex.getMessage(),
                LocalDateTime.now()
        );
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(error);
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidation(MethodArgumentNotValidException ex) {
        String message = ex.getBindingResult()
                .getFieldErrors()
                .stream()
                .map(error -> error.getField() + ": " + error.getDefaultMessage())
                .collect(Collectors.joining(", "));

        ErrorResponse error = new ErrorResponse(
                HttpStatus.BAD_REQUEST.value(),
                message,
                LocalDateTime.now()
        );
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(error);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGeneral(Exception ex) {
        ErrorResponse error = new ErrorResponse(
                HttpStatus.INTERNAL_SERVER_ERROR.value(),
                "Internal Server Error",
                LocalDateTime.now()
        );
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(error);
    }
}
```

Output khi resource không tìm thấy:
```json
{
    "status": 404,
    "message": "User not found with id: 99",
    "timestamp": "2024-04-15T10:30:00"
}
```

## 6. Bean Validation

### 6.1. Validation Annotations phổ biến

Thêm dependency:
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-validation</artifactId>
</dependency>
```

| Annotation | Mô tả |
| ---------- | ----- |
| `@NotNull` | Không null |
| `@NotBlank` | Không null, không rỗng, không chỉ chứa khoảng trắng |
| `@NotEmpty` | Không null và không rỗng |
| `@Size(min, max)` | Giới hạn kích thước |
| `@Min`, `@Max` | Giá trị tối thiểu / tối đa |
| `@Email` | Định dạng email hợp lệ |
| `@Pattern(regexp)` | Khớp với regex |
| `@Past`, `@Future` | Ngày trong quá khứ / tương lai |

```java
@Getter
@Setter
public class CreateUserRequest {

    @NotBlank(message = "Tên không được để trống")
    @Size(min = 2, max = 50, message = "Tên phải từ 2-50 ký tự")
    private String name;

    @NotBlank(message = "Email không được để trống")
    @Email(message = "Email không hợp lệ")
    private String email;

    @NotBlank(message = "Mật khẩu không được để trống")
    @Size(min = 6, message = "Mật khẩu tối thiểu 6 ký tự")
    private String password;

    @Min(value = 18, message = "Tuổi tối thiểu là 18")
    @Max(value = 100, message = "Tuổi tối đa là 100")
    private int age;
}
```

Kích hoạt validation bằng `@Valid`:
```java
@PostMapping("/api/users")
public ResponseEntity<User> createUser(@RequestBody @Valid CreateUserRequest request) {
    // Nếu validation fail → MethodArgumentNotValidException
    return ResponseEntity.status(HttpStatus.CREATED)
            .body(userService.create(request));
}
```

## 7. Câu hỏi phỏng vấn thường gặp

__Q: `@Controller` và `@RestController` khác nhau như thế nào?__

`@RestController` = `@Controller` + `@ResponseBody`. `@Controller` trả về view name (HTML), `@RestController` trả về object được serialize thành JSON/XML.

__Q: `@RequestParam` vs `@PathVariable`?__

`@PathVariable` lấy giá trị từ URL path (`/users/{id}`), `@RequestParam` lấy từ query parameters (`/users?id=1`).

__Q: Làm sao xử lý exception trong Spring Boot?__

Sử dụng `@RestControllerAdvice` + `@ExceptionHandler` để xử lý tập trung. Tạo các custom exception class và ErrorResponse DTO.

__Q: `@RequestBody` hoạt động như thế nào?__

`@RequestBody` sử dụng __HttpMessageConverter__ (mặc định là Jackson) để tự động deserialize JSON/XML trong request body thành Java object.
